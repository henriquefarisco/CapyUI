# v0.3.0 — Foco e travessia por teclado

**Status:** delivered (2026-05-19)
**ABI bump:** capy-ui-widget 0.2 → 0.3
**Display-list schema:** 1 → 2 (`CAPY_DL_FOCUS_RING`)
**Dependência:** v0.2.0 entregue
**Estimativa:** M
**Owner:** Cascade

## Objetivo

Introduzir modelo determinístico de foco com Tab/Shift+Tab, sem depender de input driver real. Permite navegação completa por teclado e habilita app sem mouse.

## Escopo CapyUI

- Campos novos em `struct capy_widget` (aditivos, no final):
  - `uint8_t focusable;`
  - `int16_t tab_index;`
- APIs públicas:
  - `void capy_widget_set_focusable(struct capy_widget *w, int focusable, int16_t tab_index);`
  - `struct capy_widget *capy_widget_focus_next(struct capy_widget *root, struct capy_widget *current);`
  - `struct capy_widget *capy_widget_focus_prev(struct capy_widget *root, struct capy_widget *current);`
  - `int capy_widget_dispatch_key(struct capy_widget *root, const struct capy_widget_event *ev);`
- `dispatch_key` consome: `TAB`, `SHIFT+TAB`, `ENTER`, `SPACE`, `ESC`, setas direcionais.
- Defaults por tipo:
  - Focáveis por default: `BUTTON`, `TEXTBOX`, `CHECKBOX`, `LIST`, `MENU_ITEM`, `TABS`.
  - Não focáveis: `LABEL`, `PANEL`, `PROGRESS`, `SCROLLBAR` (mas filhos podem ser).
- Display-list ganha op aditiva `CAPY_DL_FOCUS_RING` (schema → 2).
- Widget marcado como focado emite o ring após border, antes de filhos.

## Escopo CapyOS

Nenhum. Núcleo 100% testável em host.

## Testes obrigatórios

1. **Ordem por `tab_index`:** widgets com tab_index crescente; verificar sequência.
2. **Desempate por inserção:** dois widgets com mesmo tab_index seguem ordem de `add_child`.
3. **Skip de não-focáveis:** `enabled=0`, `visible=0`, `focusable=0` são pulados.
4. **Wrap-around:** após último, `focus_next` retorna primeiro.
5. **Reverse wrap-around:** antes do primeiro, `focus_prev` retorna último.
6. **`ENTER` em BUTTON focado dispara `on_click`.**
7. **`SPACE` em CHECKBOX focado alterna `checked` + dispara `on_change`.**
8. **`ESC` em DIALOG focado dispara callback de cancelamento.**
9. **Display-list inclui `FOCUS_RING` quando há widget focado.**
10. **Determinismo:** mesmo input de eventos → mesmo widget focado.
