# v0.4.0 — Modelo de edição de texto

**Status:** delivered (2026-05-19)
**ABI bump:** capy-ui-widget 0.3 → 0.4
**Display-list schema:** 2 (sem mudança; password e caret usam `TEXT`/`RECT` existentes)
**Dependência:** v0.3.0 entregue (foco habilita edição)
**Estimativa:** L
**Owner:** Cascade

## Objetivo

`TEXTBOX` ganha caret, seleção, edição (insert/delete), modos password e readonly, stub de IME. Tudo determinístico, sem IME real (que entrará em `v1.8.0`).

## Escopo CapyUI

- Struct aditiva embutida em `capy_widget` quando `type == TEXTBOX`:
  ```c
  struct capy_text_edit {
    uint16_t caret;
    uint16_t sel_start;
    uint16_t sel_end;
    uint8_t multiline;
    uint8_t readonly;
    uint8_t password;
    uint8_t reserved;
  };
  ```
- APIs públicas:
  - `int  capy_textbox_insert(struct capy_widget *w, const char *utf8, uint16_t len);`
  - `int  capy_textbox_delete(struct capy_widget *w, int16_t count);` (negativo = backspace)
  - `void capy_textbox_set_selection(struct capy_widget *w, uint16_t start, uint16_t end);`
  - `int  capy_textbox_copy(struct capy_widget *w, char *out, uint16_t out_cap);`
  - `int  capy_widget_ime_compose(struct capy_widget *w, const char *preedit, uint16_t len);` (stub, apenas registra estado)
- Validação UTF-8 mínima: rejeita sequências mal-formadas em `insert`.
- `password` mode emite `*` no display-list em vez do texto real.
- `readonly` mode bloqueia mutações mas permite `copy`.
- Renderização de caret no display-list: pequeno retângulo após cmd de texto.

## Escopo CapyOS

Nenhum. Host real fornecerá IME completo em `v1.8.0`.

## Testes obrigatórios

1. **Insert no meio de string preserva UTF-8** (caracteres multibyte intactos).
2. **Insert > capacidade do widget retorna erro, sem corrupção.**
3. **Selection + delete substitui região selecionada.**
4. **Caret avança após insert.**
5. **Backspace via `delete(-1)` remove caractere anterior em UTF-8 correto.**
6. **Password mode: display-list emite `*` repetido com mesmo tamanho.**
7. **Readonly: `insert` retorna erro; `copy` funciona.**
8. **IME stub: `ime_compose` armazena preedit; texto principal não muda até commit.**
9. **Determinismo:** sequência de inserts/deletes → mesma string final.
10. **UTF-8 inválido rejeitado:** `insert("\xff\xff", 2)` retorna erro.
