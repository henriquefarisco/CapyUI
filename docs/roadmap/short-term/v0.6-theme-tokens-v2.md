# v0.6.0 — Tokens de tema v2 + alto contraste

**Status:** delivered (2026-05-19) — **fecha fase de curto prazo**
**ABI bump:** capy-ui-widget 0.5 → 0.6
**Display-list schema:** 2 (sem mudança; theme é ponteiro opcional, não nova op)
**Dependência:** v0.0.1 (apenas core)
**Estimativa:** M
**Owner:** Cascade

> Nota de implementação: `CAPY_TOKEN_NONE = 0` foi adicionado ao enum para garantir backward-compat sem ambiguidade (style com tokens zero-init usa literais). `capy_theme_resolve` exposta como API pública defensiva (lida com NULL theme / token=0 / out-of-range). Theme é ponteiro aditivo em `capy_display_list`, não nova op no schema.

## Objetivo

Desacoplar cores de literais via sistema de tokens; suportar variantes light, dark e high-contrast. Permite trocar tema sem rebuild da árvore.

## Escopo CapyUI

- Enum aditivo (extensível):
  ```c
  enum capy_token {
    CAPY_TOKEN_BG_BASE = 0,
    CAPY_TOKEN_BG_RAISED,
    CAPY_TOKEN_BG_SUNKEN,
    CAPY_TOKEN_FG_PRIMARY,
    CAPY_TOKEN_FG_MUTED,
    CAPY_TOKEN_FG_INVERSE,
    CAPY_TOKEN_ACCENT,
    CAPY_TOKEN_ACCENT_HOVER,
    CAPY_TOKEN_BORDER,
    CAPY_TOKEN_BORDER_FOCUS,
    CAPY_TOKEN_FOCUS_RING,
    CAPY_TOKEN_DANGER,
    CAPY_TOKEN_WARNING,
    CAPY_TOKEN_SUCCESS,
    CAPY_TOKEN_INFO,
    CAPY_TOKEN_DISABLED,
    CAPY_TOKEN_COUNT
  };
  ```
- Struct:
  ```c
  struct capy_theme {
    uint32_t tokens[CAPY_TOKEN_COUNT];
    uint8_t variant;          /* light=0, dark=1, high_contrast=2 */
    uint8_t high_contrast;
    uint16_t version;
  };
  ```
- APIs:
  - `void capy_theme_apply(struct capy_widget_context *ctx, const struct capy_theme *theme);`
  - `struct capy_theme capy_theme_default_light(void);`
  - `struct capy_theme capy_theme_default_dark(void);`
  - `struct capy_theme capy_theme_high_contrast(void);`
- `capy_widget_style` ganha campos aditivos `uint8_t bg_token`, `fg_token`, `border_token` (0 = use literal `bg_color` etc.).
- Style resolution: se `*_token != 0`, usa `ctx->theme.tokens[token]`; caso contrário literal.

## Escopo CapyOS

Nenhum nesta entrega. Em `v0.7.0` o adapter passará a fornecer tema do SO via `capy_theme_apply`.

## Testes obrigatórios

1. **Troca de tema reemite display-list** com cores novas, mas mesma estrutura (mesma sequência de ops).
2. **High-contrast garante contraste WCAG AA** numericamente (verificar pares fg/bg).
3. **Backward compat:** widget com `style.bg_color = 0xFFRRGGBB` e `bg_token = 0` continua funcionando.
4. **`theme_default_light` e `theme_default_dark` produzem valores documentados** (snapshot).
5. **Token fora do enum** trata como literal (não crash).
6. **Determinismo:** mesma árvore + mesmo tema → mesma display-list byte-a-byte.

## Notas de compatibilidade

- Aditivo total. Programas que usam literais (`bg_color = 0xFF...`) continuam.
- Style ganha 3 campos `uint8_t` no final.
- Para programas modernos: setar `bg_token = CAPY_TOKEN_BG_BASE` e deixar tema controlar.
- Token enum cresce apenas no final pré-1.0.
