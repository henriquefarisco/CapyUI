**Display-list schema:** 2 (sem mudança; animação opera fora do display-list)
**Dependência:** v0.0.1 (apenas core); entregue em paralelo com v0.3/v0.4

> Nota de implementação: o fixed-point efetivamente entregue foi **16.16** (não 24.8 como rascunhado), com intermediários `uint64_t`/`int64_t` para evitar overflow no produto `delta * eased`. Comportamento determinístico e endpoints exatos preservados.

## Objetivo

Modelo de animação determinístico baseado em **ticks discretos**, sem clock real e sem float. Host fornece `now` monotônico em ms.

## Escopo CapyUI

- Struct pública:
  ```c
  struct capy_anim {
    uint32_t start_tick;
    uint32_t duration_ticks;
    int32_t from;
    int32_t to;
    uint8_t easing;
    uint8_t active;
    uint16_t reserved;
  };
  ```
- Enum easings:
  ```c
  enum capy_anim_easing {
    CAPY_ANIM_LINEAR = 0,
    CAPY_ANIM_EASE_IN,
    CAPY_ANIM_EASE_OUT,
    CAPY_ANIM_EASE_IN_OUT
  };
  ```
- APIs:
  - `void capy_anim_start(struct capy_anim *a, uint32_t now, uint32_t duration, int32_t from, int32_t to, uint8_t easing);`
  - `int  capy_anim_sample(const struct capy_anim *a, uint32_t now, int32_t *out);`
  - `void capy_widget_tick(struct capy_widget *root, uint32_t now);`
- Easings implementadas em **inteiros** (fixed-point 24.8 internamente; sem float):
  - `LINEAR`: `from + (to-from) * t / duration`.
  - `EASE_IN`: `t^2 / duration` (curva quadrática).
  - `EASE_OUT`: `1 - (1-t)^2`.
  - `EASE_IN_OUT`: combinação simétrica.
- `widget_tick` percorre árvore e atualiza animações ativas em campos visuais (cor, posição, alpha futuro).

## Escopo CapyOS

Host fornece `now` (uint32_t ms monotônico). Sem requisito adicional.

## Testes obrigatórios

1. **Sample em `now == start_tick`** retorna `from`.
2. **Sample em `now >= start + duration`** retorna `to`.
3. **Sample em meio do intervalo** retorna valor interpolado conforme easing.
4. **Easings monotônicas** em range `[from, to]` ou `[to, from]`.
5. **Animação não-ativa** retorna `from` sempre.
6. **Determinismo:** mesma sequência de `(now, anim)` → mesma sequência de outputs.
7. **`tick` com `now` retrocedido** (clock pulou para trás): comportamento estável, não corrompe estado.
8. **Múltiplas animações simultâneas em widgets diferentes** não interferem.

## Notas de compatibilidade

- Sem dependência de `<math.h>`: tudo em inteiros (`int32_t`).
- Range de `from`/`to`: `int32_t` permite cores RGBA, coordenadas, qualquer valor inteiro.
- Animação **não toca em display-list schema**: widgets atualizam cores/posições antes de `capy_widget_emit`.
- Pós-`v1.3.0` (rich animation), tracks e keyframes virão como extensão aditiva.
