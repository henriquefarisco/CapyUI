## Regras

- **Pré-1.0:** aditivo apenas (em ambas as ABIs). Ver `dependencies/breaking-change-policy.md`.
- **1.0 → 2.0:** deprecação ≥2 minor releases antes de remoção. Guia de migração obrigatório.
- **2.x LTS:** ≥12 meses pós-3.0.
- Releases de **packaging** (sem mudança de API) **não** bumpam ABI; apenas patch/minor do pacote.
