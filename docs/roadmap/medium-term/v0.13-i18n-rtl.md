## Objetivo

Suporte a RTL (right-to-left), locale, plurais e formatação de datas/números.

## Escopo CapyUI

- Struct em `capy_widget_context`:
  ```c
  struct capy_locale {
    char tag[16];           /* BCP 47: "en-US", "pt-BR", "ar-EG" */
    uint8_t rtl;            /* 1 se layout RTL */
    uint8_t plural_rule;    /* enum capy_plural_rule */
    uint16_t reserved;
  };
  ```
- Enum `capy_plural_rule`:
  - `CAPY_PLURAL_EN` (one, other)
  - `CAPY_PLURAL_PT` (one, other)
  - `CAPY_PLURAL_AR` (zero, one, two, few, many, other)
  - `CAPY_PLURAL_RU` (one, few, many, other)
  - `CAPY_PLURAL_OTHER` (apenas other)
- API:
  - `void capy_context_set_locale(struct capy_widget_context *ctx, const struct capy_locale *l);`
  - `int  capy_locale_format(const struct capy_locale *l, const char *fmt, ..., char *out, uint16_t out_cap);` (subconjunto de printf: `%d`, `%s`, `%u`, `%x`)
  - `int  capy_locale_plural(const struct capy_locale *l, uint32_t n);` (retorna índice da forma plural)
- Layout reflui automaticamente quando `rtl=1`:
  - `CAPY_LAYOUT_STACK` horizontal inverte ordem visual dos filhos.
  - `padding_left/right` trocados em rendering.

## Escopo CapyOS

- Detecção de locale do SO; push via adapter para o contexto CapyUI.
- Catálogos `.mo`/`.po` ou formato próprio para tradução de strings.
- IME RTL adequado em `v1.8.0`.

## Testes obrigatórios

1. **Snapshot RTL inverte posições horizontais** em STACK horizontal; verticais inalteradas.
2. **Plurais `en/pt/ar/ru`** resolvem corretamente para `0, 1, 2, 5, 21, 100`.
3. **`capy_locale_format("%d items")`** funciona em todas as locales.
4. **GRID e FLOW** respeitam RTL em fluxo (primeiro item no canto correto).
5. **Determinismo:** mesma locale + mesma árvore → mesmo display-list.
