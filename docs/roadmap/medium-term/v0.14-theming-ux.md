## Objetivo

Usuário final escolhe tema; CapyOS persiste; preview ao vivo no painel de configurações.

## Escopo CapyUI

- APIs de serialização canônica:
  ```c
  int capy_theme_serialize(const struct capy_theme *t,
                           char *out, uint32_t cap);
  int capy_theme_deserialize(struct capy_theme *out,
                             const char *in, uint32_t len);
  ```
- Formato canônico: key=value, uma por linha:
  ```
  version=2
  variant=dark
  high_contrast=0
  bg_base=0xFF1A1B1F
  bg_raised=0xFF24262C
  fg_primary=0xFFE8ECEF
  ...
  ```
- Determinístico: mesma `capy_theme` → mesma string byte-a-byte.
- Round-trip preserva todos os tokens.
- Tema malformado rejeitado com erro determinístico (offset e razão).

## Escopo CapyOS

- Persistência em `~/.config/capyos/theme.conf`.
- Painel de configurações com preview ao vivo (consome `capy_theme_apply`).
- Watcher de arquivo para hot reload em dev (precursor de v2.4 theme packs).

## Testes obrigatórios

1. **Round-trip:** `serialize → deserialize` preserva todos os tokens e variant.
2. **Determinismo:** mesma theme → mesma string.
3. **Tema malformado rejeitado** com offset do erro.
4. **Versão desconhecida** rejeitada (deserialize falha limpo).
5. **Tokens extras (futuros)** preservados quando schema cresce — ou ignorados controlamente.
6. **Capacity overflow** em `serialize` retorna `-1` sem corromper.
