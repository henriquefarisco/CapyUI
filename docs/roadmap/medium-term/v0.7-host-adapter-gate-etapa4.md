## Objetivo

Integração inicial entre CapyOS e CapyUI. **Só prossegue se Etapa 4 do roadmap CapyOS autorizar.**

## Escopo CapyUI

Nenhum código novo no core. Apenas:

- Exporta ABI 0.7 estável (sem mudança técnica relevante além de bump de versão).
- Atualiza `docs/compatibility.md` marcando "CapyOS adapter integrated under Etapa 4".
- Pode haver ajustes mínimos em API se feedback do adapter exigir.

## Escopo CapyOS

Trabalho real no repositório CapyOS:

- Novo módulo `CapyOS/src/ui_adapter/`.
- Mapeamentos obrigatórios:
  - `gui_event` (CapyOS) → `capy_widget_event` (pointer/key).
  - Tema do SO → `capy_theme` (variante light/dark + high contrast quando habilitado).
  - Locale do SO → preparação para v0.13.
- Roteamento de quadro:
  1. Receber input do SO.
  2. `capy_widget_dispatch_key/pointer` no root da janela.
  3. `capy_widget_tick(now)` com clock monotônico.
  4. `capy_widget_measure(root, w, h)` + `capy_widget_arrange(root)`.
  5. `capy_widget_emit(root, &dl)`.
  6. Render display-list no compositor CapyOS (CPU rasterizer inicial).
- **Sem inclusão** de estruturas CapyOS dentro de CapyUI (regra inegociável).

## Testes obrigatórios (lado CapyOS)

1. **Fixture de input do SO** produz mesmas transições do modelo retido testado em host.
2. **Display-list renderiza pixel-equivalente** a fixture canônica em host.
3. **Rollback:** desinstalar adapter restaura GUI legada CapyOS sem crash.
4. **Smoke `adapter-roundtrip-vmware`:** janela aparece, recebe clique, fecha.

## Release gate

- **Combinado:** `make validate` em CapyUI + suíte de integração em CapyOS + validação manual de Etapa 4.
- Auditoria estática: `grep -rn '#include <gui/' src/widget/` deve retornar vazio.

## Notas de compatibilidade

- ABI 0.7 é aditivo apenas em metadata; nenhuma API nova exposta diretamente.
- Adapter usa apenas APIs ≥ 0.0 que já estão estáveis.
- Display-list schema permanece 2.
- A partir deste ponto, releases de CapyUI requerem coordenação com CapyOS para validar integração.
