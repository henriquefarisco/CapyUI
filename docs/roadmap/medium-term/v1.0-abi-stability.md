# v1.0.0 — Estabilidade do ABI

**Status:** planned
**ABI bump:** capy-ui-widget 0.15 → **1.0 (CONGELADO)**
**Display-list schema:** 4 (sem mudança nesta release)
**Dependência:** v0.15.0 entregue + 30 dias contínuos de CI verde
**Estimativa:** M (consolidação, não código novo)
**Owner:** —

## Objetivo

Congelar `capy-ui-widget` 1.x. Marca início da política de deprecação formal. Mudanças pós-1.0 são aditivas ou em nova major com guia de migração.

## Escopo CapyUI

- **Sem código novo.** Apenas consolidação:
  - `VERSION` → `1.0.0`.
  - `README.md` → `Version: 1.0.0`.
  - `docs/compatibility.md` → ABI marcado como `1.0 (stable)`.
  - Auditoria final de headers públicos: nenhum `internal_*` exposto.
  - Headers ganham comentário `/* @since 0.X.Y */` em cada símbolo público.
  - Macro `CAPYUI_API_DEPRECATED` introduzida (placeholder, ainda não usada).
- `SECURITY.md` atualizado com política de divulgação responsável.
- LICENSE auditada.

## Critérios de saída

1. **Todas as fatias v0.1–v0.15 com testes verdes** em CI por **30 dias contínuos**.
2. `docs/compatibility.md` consolidado com:
   - Matriz de compatibilidade CapyOS × CapyUI (final).
   - Política de deprecação formalizada (cite `dependencies/breaking-change-policy.md`).
   - Procedimento de rollback documentado.
3. Auditoria externa: **nenhum include direto de runtime CapyOS** detectado em CapyUI.
4. `VERSION` → `1.0.0`; `README.md` atualizado; release tag `v1.0.0` no GitHub assinada.
5. `tracking/CHANGELOG.md` entrada destacada para v1.0.0 com guia de migração de 0.x.
6. Comunicado público (release notes) com:
   - Garantias de compatibilidade.
   - Roadmap pós-1.0 (linka `long-term/`).
   - Como reportar regressões.

## Escopo CapyOS

- CapyOS deve estar em Etapa adequada para consumir CapyUI 1.0 (mínimo Etapa 6 nova ordenação).
- Validação cruzada: rodar app CapyOS contra CapyUI 1.0 sem regressão.

## Testes obrigatórios

- Todos os testes anteriores (v0.0.1–v0.15) continuam verdes.
- **Auditoria de ABI:** diff de headers vs. v0.15.0 deve ser apenas aditivo.
- **Audit de inclusão:** `grep` por includes proibidos retorna vazio.
- **Soak test:** suite roda 1000× consecutivas em CI sem flakiness.

## Contratos atualizados

- `contracts/abi-versions.md` (1.0 marcado como CONGELADO).
- `contracts/deprecation-policy.md` (regras formais em vigor).

## Release gate

- `make validate` em CapyUI verde.
- Suíte de integração CapyOS verde.
- Aprovação manual em Etapas 4 e 6 já consolidadas.
- Anúncio público com guia de migração 0.x → 1.x.

## Notas de compatibilidade

- **Esta é a versão de referência.** A partir daqui, qualquer quebra exige:
  - ADR em `tracking/DECISIONS.md`.
  - Deprecação por ≥2 minor releases.
  - Major bump (`2.0.0`) para remoção.
- LTS de 1.x: **≥12 meses** após release de 2.0.
- ABI 0.x continua suportado em CapyOS via adapter de compatibilidade (host adapter problem, não CapyUI).
