# v0.9.0 — Texto e fontes host

**Status:** planned
**ABI bump:** capy-ui-widget 0.8 → 0.9
**Display-list schema:** 3 → 4 (`font_id` em TEXT)
**Dependência:** v0.8.0 entregue
**Estimativa:** M (CapyUI) + L (CapyOS)
**Owner:** —

## Objetivo

Shaping e rasterização real de texto via host CapyOS; CapyUI continua emitindo apenas referências (`font_id`, `text_offset`, `text_len`).

## Escopo CapyUI

- Op `CAPY_DL_TEXT` ganha campo aditivo `uint16_t font_id` (preenchido por `0` = fonte default).
- Display-list schema → 4 (aditivo, ignorado por consumidores antigos).
- API opcional do hook de métricas:
  ```c
  typedef int (*capy_text_metrics_fn)(
      uint16_t font_id, uint8_t font_size,
      const char *text, uint16_t len,
      uint32_t *out_width, uint32_t *out_height,
      void *user_data);

  void capy_widget_set_text_metrics_hook(
      struct capy_widget_context *ctx,
      capy_text_metrics_fn fn,
      void *user_data);
  ```
- Quando hook **não está setado**, fallback determinístico:
  - `width = len * font_size / 2`
  - `height = font_size`
- Quando hook está setado, `capy_widget_measure` o chama em widgets com texto.

## Escopo CapyOS

- Font loader (TTF/OTF).
- Shaper (HarfBuzz-like ou subset próprio).
- Cache de glifos.
- Adapter chama `capy_widget_set_text_metrics_hook` no init e fornece função real.

## Testes obrigatórios

1. **Sem hook:** layout usa fallback determinístico documentado.
2. **Com hook do host:** layout reflui quando fonte muda.
3. **`font_id = 0` (default)** sempre válido.
4. **Display-list emite `font_id`** corretamente em todos os TEXT.
5. **Determinismo do fallback:** mesma string + mesmo font_size → mesma largura.

## Notas de compatibilidade

- `font_id` é aditivo em payload de TEXT; struct `capy_dl_cmd` cresce no campo `reserved` já existente (será dividido).
- Fallback garante que testes host puros continuam funcionando.
- Schema 3 vs. 4: consumidor lê `font_id` apenas se `dl->version >= 4`.
