## Objetivo

Árvore de acessibilidade espelhando o modelo retido. Permite screen readers, magnifier e atalhos globais.

## Escopo CapyUI

- Struct pública:
  ```c
  struct capy_a11y_node {
    uint32_t widget_id;      /* hash estável do widget */
    const char *role;        /* "button", "checkbox", etc. */
    const char *label;       /* texto descritivo */
    const char *description; /* descrição longa opcional */
    uint32_t state_flags;    /* focused, checked, disabled, etc. */
    uint32_t parent_id;      /* hierarquia espelhada */
  };
  ```
- API:
  ```c
  int capy_widget_a11y_snapshot(struct capy_widget *root,
                                struct capy_a11y_node *out,
                                uint32_t cap);
  ```
- Roles obrigatórios:
  - `button`, `checkbox`, `textbox`, `menu`, `menuitem`, `list`, `listitem`,
    `tab`, `tabpanel`, `progressbar`, `dialog`, `label` (para LABEL widget).
- State flags (bitmask):
  - `CAPY_A11Y_FOCUSED`, `CAPY_A11Y_CHECKED`, `CAPY_A11Y_DISABLED`,
    `CAPY_A11Y_HIDDEN`, `CAPY_A11Y_PRESSED`, `CAPY_A11Y_EXPANDED`.

## Escopo CapyOS

- Bridge para API de acessibilidade do SO (screen reader, magnifier).
- Atalhos globais (`Win+U` para central de acessibilidade, etc.).
- Eventos do bridge: focar widget por nome, ler árvore, navegar headings.

## Testes obrigatórios

1. **Snapshot estável:** árvore não muda sem mutação de modelo.
2. **Roles cobrem 100% dos tipos focáveis.**
3. **`widget_id` é estável** entre snapshots (hash não muda se widget não muda).
4. **State flags refletem estado real** (focused/checked/disabled).
5. **Overflow de `cap`** retorna `-1` sem corromper estado.
6. **`label`** segue a regra: 1º `text`, 2º `aria-label` (futuro), 3º "Untitled".
7. **Determinismo:** mesma árvore → mesmo snapshot byte-a-byte.
