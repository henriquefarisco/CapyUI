# v1.6.0 — Drag and drop

**Status:** delivered (CapyUI side, 2026-05-21). Modelo de dados + roteamento de DROP no widget core. DRAG_BEGIN/MOVE/END são pass-through (no-op) — host dispatcher possui o ciclo de preview/cancel. Display-list dedicado para preview deferido para fatia futura.
**ABI bump:** capy-ui-widget 1.5 → 1.6 (aditivo)
**Display-list schema:** permanece **5** (gesture/DnD não emitem novas ops; o `6` do plano original assumia bumps anteriores que não ocorreram).
**Dependência:** v1.5.0 entregue
**Estimativa:** M (CapyUI ✅)
**Owner:** CapyUI

## Objetivo

Modelo de drag and drop com payloads tipados (MIME-like sem deps), drop targets com filtros, e preview via display-list dedicado.

## Escopo CapyUI

- API:
  ```c
  struct capy_dnd_payload {
    char type[32];          /* "text/plain", "app/widget-id", etc. */
    const void *data;
    uint32_t len;
  };

  void capy_widget_set_draggable(struct capy_widget *w,
                                 const struct capy_dnd_payload *payload);
  void capy_widget_set_drop_target(struct capy_widget *w,
                                   const char *accepted_types[],
                                   uint32_t count);

  typedef int (*capy_dnd_callback_fn)(
      struct capy_widget *target,
      const struct capy_dnd_payload *payload,
      void *user_data);

  void capy_widget_set_on_drop(struct capy_widget *w,
                               capy_dnd_callback_fn fn,
                               void *user_data);
  ```
- Eventos abstratos novos:
  - `CAPY_WIDGET_EVENT_DRAG_BEGIN`
  - `CAPY_WIDGET_EVENT_DRAG_MOVE`
  - `CAPY_WIDGET_EVENT_DRAG_END`
  - `CAPY_WIDGET_EVENT_DROP`
- Display-list preview: durante drag, widget origem emite preview translúcido em posição do cursor (compositor renderiza on-top).

## Testes que cobrem o contrato

- `test_dnd_set_draggable_attaches_payload` — setter + clear + NULL guard.
- `test_dnd_set_drop_target_attaches_filter` — setter; NULL array clampa count a zero; NULL widget no-op.
- `test_dnd_type_match_exact_wildcard_prefix` — exact (case-insensitive), `text/*` prefix, `*` wildcard, mismatched lengths, NULL guards.
- `test_dnd_drop_invokes_callback_when_compatible` — DROP em `text/plain` com filter `text/*` dispara callback com payload, target e user_data corretos.
- `test_dnd_drop_silently_ignored_when_incompatible` — DROP em `image/png` com filter `text/*` não dispara; DROP fora de bounds também não dispara.
- `test_dnd_event_types_routed_without_crash` — DRAG_BEGIN/MOVE/END são no-ops; DROP com NULL payload não crasha.
- `test_dnd_dnd_accepts_helper` — helper público retorna 1/0 corretamente; NULL guards.
- `test_dnd_drop_deepest_first` — outer e inner panels ambos são drop targets; drop dentro de inner roteia para inner (deepest-first), não outer.

**Não testado em 1.6** (deferido):
- Cancel via ESC durante drag — sem stado de drag persistente no core; o host dispatcher implementa cancel via DRAG_END.
- Determinismo end-to-end com host preview — fora do escopo do core.

## Notas

- `struct capy_dnd_payload` tem `type[32]` por valor (zero-alloc) + `data` ponteiro + `len`. Payloads passam sempre por ponteiro `const struct capy_dnd_payload *`; CapyUI nunca copia nem libera.
- Sem cópia interna de `data` no core; lifecycle responsabilidade do caller. O array `drop_accepted_types` também é caller-owned.
- Host CapyOS pode mediar drag entre apps via clipboard, mas core apenas roteia DROP.
- DRAG_BEGIN/MOVE/END são silenciosamente aceitos no core (return 0). O host dispatcher carrega o stado de drag entre BEGIN e END (cursor pos, preview, ESC para cancelar). Quando o usuário solta, o dispatcher sintetiza um `CAPY_WIDGET_EVENT_DROP` com `event->payload = const struct capy_dnd_payload *` e o core roteia para o deepest accept.
- Pattern language minimalista: `"*"` (wildcard puro), `"prefix/*"` (prefix), exact. Sem char classes, sem regex. Case-insensitive em ASCII; bytes UTF-8 acima de 0x7F comparados literalmente.
- Preview no display-list deferido: pode entrar como op aditiva futura (ex.: `CAPY_DL_DRAG_PREVIEW`) sem bump de major.
