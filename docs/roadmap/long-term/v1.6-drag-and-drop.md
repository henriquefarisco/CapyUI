# v1.6.0 — Drag and drop

**Status:** planned
**ABI bump:** capy-ui-widget 1.5 → 1.6 (aditivo)
**Display-list schema:** 6 (sem mudança)
**Dependência:** v1.5.0 entregue
**Estimativa:** M
**Owner:** —

## Objetivo

Modelo de drag and drop com payloads tipados (MIME-like sem deps), drop targets com filtros, e preview via display-list dedicado.

## Escopo CapyUI

- API:
  ```c
  struct capy_dnd_payload {
    char type[32];          /* "text/plain", "app/widget-id", etc. */
    const void *data;
    uint32_t len;
  };

  void capy_widget_set_draggable(struct capy_widget *w,
                                 const struct capy_dnd_payload *payload);
  void capy_widget_set_drop_target(struct capy_widget *w,
                                   const char *accepted_types[],
                                   uint32_t count);

  typedef int (*capy_dnd_callback_fn)(
      struct capy_widget *target,
      const struct capy_dnd_payload *payload,
      void *user_data);

  void capy_widget_set_on_drop(struct capy_widget *w,
                               capy_dnd_callback_fn fn,
                               void *user_data);
  ```
- Eventos abstratos novos:
  - `CAPY_WIDGET_EVENT_DRAG_BEGIN`
  - `CAPY_WIDGET_EVENT_DRAG_MOVE`
  - `CAPY_WIDGET_EVENT_DRAG_END`
  - `CAPY_WIDGET_EVENT_DROP`
- Display-list preview: durante drag, widget origem emite preview translúcido em posição do cursor (compositor renderiza on-top).

## Testes obrigatórios

1. **Drag de widget A para drop target B compatível** dispara callback.
2. **Drop em target incompatível** rejeitado silenciosamente.
3. **Cancel (ESC durante drag)** restaura estado.
4. **Payload type matching** caseinsensitive em prefixo (`text/*` aceita `text/plain`).
5. **Determinismo:** mesma sequência → mesmo callback chamado.

## Notas

- Payloads pequenos passam por valor; grandes via ponteiro (caller mantém vivo).
- Sem cópia interna de `data` no core; lifecycle responsabilidade do caller.
- Host CapyOS pode mediar drag entre apps via clipboard, mas core apenas roteia.
