# v2.1.0 — Widgets avançados

**Status:** delivered (CapyUI side, 2026-05-21) com **escopo conservador**: 8 enum slots reservados (`TREE`/`TABLE`/`RICH_TEXT`/`CANVAS`/`CHART`/`COLOR_PICKER`/`DATE_PICKER`/`AUTOCOMPLETE`). Cada tipo integra com o pipeline genérico (layout, emit, focus, serialize, a11y) usando os campos existentes do `capy_widget`. **Per-widget state e APIs dedicadas (tree hierarchy, table columns, rich-text ranges, canvas draw callback, chart datasets, color/date pickers, autocomplete suggestions) ficam para fatias futuras 2.x aditivas**, cada uma adicionando seus próprios tail fields + APIs.
**ABI bump:** capy-ui-widget 2.0 → 2.1 (aditivo dentro do major 2)
**Display-list schema:** permanece **7** (novos tipos não exigem ops novas; cada widget emite via o pipeline genérico).
**Dependência:** v2.0.0 entregue
**Estimativa:** XL (CapyUI ✅ — escopo enum-only; APIs dedicadas são follow-ups)
**Owner:** CapyUI

## Objetivo

Conjunto rico de widgets para apps modernos.

## Escopo CapyUI

Tipos aditivos no enum `capy_widget_type`:

- **`TREE`:** hierarquia expansível com chevron, indentação, lazy load via callback.
- **`TABLE`:** linhas/colunas, header sticky, sort por header (callback), seleção single/multi.
- **`RICH_TEXT`:** ranges de estilo (cor, peso, tamanho, link) no mesmo widget. Caret e seleção integrados (reusa v0.4 text_edit).
- **`CANVAS`:** draw custom via callback que recebe display-list para preencher.
- **`CHART`:** line/bar/pie básico com dados via callback. Sem analítica complexa (delegada a apps).
- **`COLOR_PICKER`:** HSV + hex + recent. Emite evento ao mudar.
- **`DATE_PICKER`:** calendário com locale (v0.13).
- **`AUTOCOMPLETE`:** TEXTBOX + dropdown filtrado via callback.

APIs específicas por widget documentadas em headers individuais.

## Testes que cobrem o contrato (escopo enum-only)

- `test_advanced_widget_tree_creates_and_lays_out` — cria, seta bounds + text, valida campos.
- `test_advanced_widget_table_in_panel_emits` — table como filho de panel; `capy_widget_emit` produz ≥2 cmds.
- `test_advanced_widget_rich_text_basic` — cria, seta bounds + text "Hello, world!", valida bytes.
- `test_advanced_widget_canvas_creates` — cria, defaults `visible=enabled=1`.
- `test_advanced_widget_chart_creates` — cria com dataset vazio (sem crash); `set_visible` alterna.
- `test_advanced_widget_color_picker_focusable` — host marca como focusable + tab_index; campos persistem.
- `test_advanced_widget_date_picker_respects_rtl_context` — cria com `ctx.locale.rtl = 1` (since 0.13); contexto observável.
- `test_advanced_widget_autocomplete_roundtrips_serialization` — serialize ctx_a + deserialize ctx_b preserva type/bounds/text; valida enum ordering (`TREE > TABS`, `AUTOCOMPLETE > DATE_PICKER`).

**Não testado em 2.1** (deferido para 2.x minors aditivas dedicadas):
- TREE hierarchy + chevron + lazy-load callback.
- TABLE columns/header sticky/sort/seleção single/multi.
- RICH_TEXT ranges (color/weight/size/link); o slice doc original previa validação de overlap.
- CANVAS draw callback recebendo display-list.
- CHART datasets + line/bar/pie rendering.
- COLOR_PICKER HSV/hex/recent + on_change events.
- DATE_PICKER calendar grid + RTL mirroring.
- AUTOCOMPLETE callback de suggestions + filtragem determinística.
- Role mapping para a11y_snapshot dos novos tipos (atualmente caem no role default).

## Notas

- Performance considerada em v1.7 (slab) e v2.2 (virtualization para TABLE/TREE).
- Cada widget novo pode ter contrato dedicado em `contracts/widgets/` em fatias futuras.
- **Por que enum-only nesta fatia:** os 8 tipos juntos exigiriam ≈20 tail fields novos + ≈25 APIs dedicadas. Empacotar tudo numa só fatia inflaria descontroladamente `sizeof(capy_widget)`. A escolha conservadora reserva os slots de enum agora (compatíveis com `serialize`/`deserialize` e com o pipeline genérico) e deixa o per-widget state para minors 2.x onde cada widget ganha seu próprio incremento focado de struct + APIs.
- Stub `capy_widget_ime_compose` (0.4) continua funcionando para callers mínimos de RICH_TEXT enquanto a fatia rica fica pendente.
- Macros bumpadas para 2/1/0; `CAPYUI_API_VERSION_TAG = 0x00020100`.
- `capy_widget_serialize` (since 1.10) aceita os 8 tipos novos no deserialize — o type-byte ceiling foi atualizado de `CAPY_WIDGET_TABS` para `CAPY_WIDGET_AUTOCOMPLETE`.
