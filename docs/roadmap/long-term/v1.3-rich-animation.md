## Objetivo

Timeline com keyframes, spring physics em fixed-point, easing curvas customizadas (cubic-bezier inteiro). Mantém invariante de zero float em hot paths.

## Escopo CapyUI

- Tracks de animação:
  ```c
  struct capy_anim_keyframe {
    uint32_t tick;
    int32_t value;
    uint8_t easing;       /* enum capy_anim_easing */
    uint8_t reserved;
    uint16_t reserved2;
  };

  struct capy_anim_track {
    struct capy_anim_keyframe *keyframes;
    uint32_t count;
    uint32_t capacity;
    int32_t loop;          /* 0=once, -1=infinite, N=times */
  };
  ```
- Spring physics:
  ```c
  struct capy_spring {
    int32_t target;
    int32_t velocity;     /* fixed-point 24.8 */
    int32_t position;
    uint16_t stiffness;   /* fixed-point 8.8 */
    uint16_t damping;
  };

  void capy_spring_step(struct capy_spring *s, uint32_t dt_ms);
  ```
- Cubic-bezier inteiro: 4 pontos de controle em fixed-point.
- API:
  - `capy_anim_track_sample(track, now, &out)`.
  - `capy_widget_animate_property(w, property_id, from, to, duration, easing)`.

## Testes que cobrem o contrato

- `test_anim_track_sample_keyframes_ordered` — 3 keyframes (0→1000 em t=0..100, 1000→0 em t=100..200) com easing linear; sample em t=0/50/100/150/1000 retorna valores corretos; NULL guards → -1.
- `test_anim_track_loop_infinite` — loop=-1 com 2 keyframes (período 100); sample em t=50, 150, 250 produz mesmo valor; t=100 wraps para 0.
- `test_anim_track_loop_finite_clamps` — loop=2; t=50 e t=150 mid-loops batem; t=200 (=2×period) clampa para last value.
- `test_anim_spring_converges_with_damping` — stiffness=64, damping=128 (Q8.8 ≈ 0.25 e 0.5) com target=1000; após 500 substeps de 1 ms, |position - target| < 50.
- `test_anim_spring_oscillates_undamped` — stiffness=64, damping=0; em 200 substeps a posição overshoota (> target+50) e depois volta abaixo (< target-50).
- `test_anim_bezier_monotonic` — pontos (0,200,800,1000) monotônicos → curva monotônica em 17 amostras [0, 0x10000]; endpoints exatos; clamping em t fora de [0,1].
- `test_anim_track_determinism` — mesma `(now, track)` chamada 2× sobre 20 valores de `now` espaçados → outputs idênticos.

## Notas

- Spring usa Q24.8 para velocity, Q8.8 para stiffness/damping; position e target são int32 em unidades plain. Integração por symplectic Euler em substeps de 1 ms.
- Cubic-bezier 1D (de Casteljau) sobre 4 pontos de controle int32; t é Q16.16. Hosts que precisem da forma CSS easing (P0=(0,0), P3=(1,1) com solve x→t) podem implementar Newton-Raphson em cima do `bezier_eval`.
- Memória dos keyframes é caller-owned. CapyUI não aloca nem mutá o array — a struct `capy_anim_track` apenas guarda o ponteiro e o count.
- **Sem helper `capy_widget_animate_property` em 1.3.** Esse atalho exigiria slot de animação por-widget integrado com `capy_widget_tick` e binding por property id (POS_X, POS_Y, WIDTH, HEIGHT). Fica para fatia futura aditiva.
