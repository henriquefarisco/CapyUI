## Objetivo

Timeline com keyframes, spring physics em fixed-point, easing curvas customizadas (cubic-bezier inteiro). Mantém invariante de zero float em hot paths.

## Escopo CapyUI

- Tracks de animação:
  ```c
  struct capy_anim_keyframe {
    uint32_t tick;
    int32_t value;
    uint8_t easing;       /* enum capy_anim_easing */
    uint8_t reserved;
    uint16_t reserved2;
  };

  struct capy_anim_track {
    struct capy_anim_keyframe *keyframes;
    uint32_t count;
    uint32_t capacity;
    int32_t loop;          /* 0=once, -1=infinite, N=times */
  };
  ```
- Spring physics:
  ```c
  struct capy_spring {
    int32_t target;
    int32_t velocity;     /* fixed-point 24.8 */
    int32_t position;
    uint16_t stiffness;   /* fixed-point 8.8 */
    uint16_t damping;
  };

  void capy_spring_step(struct capy_spring *s, uint32_t dt_ms);
  ```
- Cubic-bezier inteiro: 4 pontos de controle em fixed-point.
- API:
  - `capy_anim_track_sample(track, now, &out)`.
  - `capy_widget_animate_property(w, property_id, from, to, duration, easing)`.

## Testes obrigatórios

1. **Keyframes ordenados por tick.**
2. **Loop infinito** wraps corretamente.
3. **Spring converge** após N steps com damping > 0.
4. **Spring com damping=0** oscila indefinidamente (sem amortecimento).
5. **Cubic-bezier monotônico** quando pontos de controle são bem-formados.
6. **Determinismo:** mesma `(now, track)` → mesmo output.

## Notas

- Spring usa fixed-point 24.8; conversão para inteiro real só na sample final.
- Cubic-bezier custom permite curvas de design (ex.: Material Design easings).
- Memória dos keyframes vem do allocator do contexto.
