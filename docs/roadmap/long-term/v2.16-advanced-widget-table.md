## Contexto

Terceira fatia da trilha de **estado dos advanced widgets** (date picker 2.14,
color picker 2.15). Diferente das duas anteriores (valores escalares), o
`TABLE` guarda um **array de larguras de coluna caller-owned** + contagem, com
acessores **bounds-checked / fail-closed**. Mantém os invariantes: zero-alloc
(CapyUI nunca copia/aloca/libera o array), zero-float, total, determinístico.

## Escopo CapyUI (entregue)

### 1. Campos de cauda

```c
const uint16_t *table_column_widths; /* caller-owned; NULL = sem modelo */
uint16_t        table_column_count;  /* nº de colunas; 0 = sem modelo */
```

Appended ao tail de `struct capy_widget`, válidos só quando
`type == CAPY_WIDGET_TABLE`. Larguras em pixels. `capy_widget_create` zera →
"sem modelo de coluna". O array é **caller-owned** e deve sobreviver ao uso
pelo widget — CapyUI só guarda o ponteiro. `sizeof(struct capy_widget)` cresce
de 824 (2.15) para 840 bytes (o ponteiro força alinhamento de 8).

### 2. APIs (total, determinístico, zero-alloc, zero-float, bounds-checked)

- `int capy_widget_set_table_columns(w, widths, count)` — armazena o modelo.
  `count == 0` limpa (widths ignorado). `count > 0` exige `widths != NULL`.
  Retorna `0`; `-1` em NULL `w` / `type != TABLE` / (`count > 0 && widths == NULL`).
  **Fail-closed:** o modelo armazenado não muda em falha.
- `int capy_widget_clear_table_columns(w)` — volta a "sem colunas" (NULL/0).
  `0`; `-1` em NULL / tipo errado.
- `int capy_widget_table_column_count(w)` — nº de colunas (0 quando nenhuma);
  `-1` em NULL / tipo errado.
- `int capy_widget_table_column_width(w, index, out_width)` — escreve
  `widths[index]` em `*out_width`. `0`; `-1` em NULL `w` / tipo errado /
  `out_width == NULL` / sem modelo / `index >= count` (bounds-check).

DL schema permanece 7: o TABLE continua emitindo pelo pipeline genérico; esta
fatia só adiciona estado + validação de índice.

## Testes obrigatórios (entregues — 8)

1. `table_set_and_query` — set {100,200,150}, count 3, leitura de cada largura.
2. `table_index_out_of_range` — `index == count` e `index` muito além → -1.
3. `table_set_zero_count_clears` — `count==0` limpa mesmo com ponteiro passado.
4. `table_set_null_widths_rejected` — `set(NULL, count>0)` → -1, modelo anterior intacto.
5. `table_clear` — set → clear → count 0, width → -1.
6. `table_wrong_widget_type_rejected` — set/clear/count/width em BUTTON → -1.
7. `table_null_guards` — NULL widget e NULL out_width → -1.
8. `table_default_no_columns` — TABLE novo → count 0, width → -1.

Total da suíte: **289 → 297** (guard de contagem pinado em `make version-check`).

## Compatibilidade

- 100% aditivo. Macros bumpadas para `2/16/0`; `CAPYUI_API_VERSION_TAG = 0x00021000`.
- `capy_widget_serialize` (1.10) **não** serializa os campos de coluna (a tabela
  é caller-owned e ponteiros não são serializáveis) — formato binário não muda.
