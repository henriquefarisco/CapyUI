# v1.9.0 — Transforms 2D

**Status:** delivered (CapyUI side, 2026-05-21). Modelo de transforms + 90°×N rotações exatas + matrix multiply em Q16.16. Arbitrary-angle rotation e hit-test compensation **deferidos**.
**ABI bump:** capy-ui-widget 1.8 → 1.9 (aditivo)
**Display-list schema:** **5 → 6** (TRANSFORM_PUSH/POP). Segundo bump real desde 0.9 (o primeiro foi 4 → 5 em v1.5).
**Dependência:** v1.8.0 entregue
**Estimativa:** M (CapyUI ✅)
**Owner:** CapyUI

## Objetivo

Transformações 2D no display-list: rotação, scale, skew, translate. Matrizes em fixed-point. Sem perspectiva (mantém determinismo).

## Escopo CapyUI

- Ops novas:
  - `CAPY_DL_TRANSFORM_PUSH`
  - `CAPY_DL_TRANSFORM_POP`
- Payload em `capy_dl_cmd` (reuso de campos existentes ou aditivo):
  ```c
  struct capy_dl_transform {
    int32_t m00, m01, m02;   /* fixed-point 16.16 */
    int32_t m10, m11, m12;
  };
  ```
- Matriz 2×3 (afim), suficiente para rotate/scale/skew/translate.
- APIs:
  ```c
  void capy_widget_set_transform(struct capy_widget *w,
                                 const struct capy_dl_transform *t);
  struct capy_dl_transform capy_transform_identity(void);
  struct capy_dl_transform capy_transform_rotate(int32_t degrees_x256);
  struct capy_dl_transform capy_transform_scale(int32_t sx_x256, int32_t sy_x256);
  struct capy_dl_transform capy_transform_translate(int32_t tx, int32_t ty);
  struct capy_dl_transform capy_transform_multiply(
      const struct capy_dl_transform *a,
      const struct capy_dl_transform *b);
  ```
- Hit-test atualizado: aplica matriz inversa ao point antes de comparar bounds.

## Testes que cobrem o contrato

- `test_transform_identity_is_diagonal` — identity matrix com m00=m11=0x10000, off-diagonals 0.
- `test_transform_scale_and_translate` — scale(2,2) tem m00=m11=0x20000; translate(5,7) tem m02=5, m12=7.
- `test_transform_rotate_quadrants_exact` — 0°/90°/180°/270° exatos; `n` reduzido modulo 4.
- `test_transform_multiply_identity_neutral` — I*X == X == X*I; NULL args caem para identity sem crash.
- `test_transform_multiply_composes` — scale×2 * translate(5,7) vs translate * scale demonstra não-comutatividade; valida métodos `m02`/`m12`.
- `test_transform_widget_emits_push_pop` — widget com transform setado emite TRANSFORM_PUSH (primeiro op, com matriz) + TRANSFORM_POP (último op); transform=NULL volta byte-equivalente a pre-1.9.
- `test_transform_gpu_translator_skips_transform_ops` — `capy_dl_to_quads` consome TRANSFORM_PUSH+RECT+TRANSFORM_POP e emite exatamente 1 quad (para o RECT); os transform ops são metadata.

**Não testado em 1.9** (deferido):
- Arbitrary-angle rotation (sin/cos LUT ou CORDIC) — hosts que precisam constroem matriz própria e passam via `set_transform`/`multiply`.
- Hit-test compensation com inverse transform — widget bounds continuam axis-aligned para `capy_widget_handle_event`. Renderer aplica a matriz; widget core não.
- Skew (shear) factory — callers populam m01/m10 manualmente.

## Notas

- 2×3 affine: rotação, scale, skew, translate. Sem perspectiva (3D real); mantemos isso fora do core.
- Fixed-point Q16.16 para a parte linear (m00/m01/m10/m11); `m02`/`m12` são integer pixels diretos (sem scaling) para convenência do caller.
- Compositor pode skip TRANSFORM_PUSH/POP (renderizar sem rotação) se backend não suporta affine — visual degradado mas funcional. GPU translator de 1.2 ignora silenciosamente.
- `multiply` usa `int64_t` intermediates + saturating clamp a `INT32_MAX/MIN`. Determinístico cross-platform.
- **Arbitrary-angle rotation adiada** para manter o core float-free e sem LUT pesado de sin/cos. Hosts que precisam de rotações não-quadrante calculam próprio (CORDIC, LUT próprio) e populam o matriz via `multiply` ou diretamente em `set_transform`.
- **Hit-test compensation adiada**: widget bounds continuam axis-aligned em `capy_widget_handle_event`. Renderer aplica a matriz visualmente; eventos chegam em coordenadas não-transformadas. Hosts que querem hit em widget rotacionado aplicam inverse transform no event antes de chamar `handle_event`.
- `sizeof(capy_dl_cmd)` cresce 24 bytes (6 × int32_t) com o novo field `transform`. Hosts que stack-allocate arrays podem precisar recompilar.
