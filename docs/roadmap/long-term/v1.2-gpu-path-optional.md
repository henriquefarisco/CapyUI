# v1.2.0 — GPU path opcional

**Status:** planned
**ABI bump:** capy-ui-widget 1.1 → 1.2 (aditivo)
**Display-list schema:** 4 → 5 (GPU primitives opcionais)
**Dependência:** v1.1.0 entregue
**Estimativa:** L
**Owner:** —

## Objetivo

Translator de display-list para primitivos GPU (quads + scissors + texture refs), sem tornar GPU obrigatório. Host CapyOS decide CPU rasterizer vs. GPU.

## Escopo CapyUI

- Novo header `src/widget/capy_dl_gpu.h` (opcional, ifdef-gated).
- API:
  ```c
  struct capy_gpu_quad {
    int32_t x0, y0, x1, y1;
    uint32_t color;
    uint32_t texture_id;     /* 0 = sem textura */
    int32_t uv_x0, uv_y0, uv_x1, uv_y1;
  };

  int capy_dl_to_quads(const struct capy_display_list *dl,
                       struct capy_gpu_quad *out, uint32_t cap,
                       uint32_t *out_count);
  ```
- Ops compostos:
  - `RECT` → 1 quad.
  - `BORDER` → 4 quads (top/right/bottom/left).
  - `TEXT` → N quads (1 por glifo, dependendo do font cache do host).
  - `IMAGE_REF` → 1 quad com texture_id.
  - `CLIP_PUSH/POP` → scissor stack do GPU backend.
- Testes: equivalência byte-a-byte entre paths CPU e GPU em fixtures.

## Testes obrigatórios

1. **Display-list simples → quads esperados** (fixtures).
2. **CLIP_PUSH/POP balanceiam scissor stack** no GPU output.
3. **Overflow de `cap`** retorna `-1` com out_count parcial.
4. **Determinismo:** mesma DL → mesma sequência de quads byte-a-byte.

## Contratos atualizados

- `contracts/display-list.md` (schema 5, GPU ops opcionais).

## Notas

- Header `capy_dl_gpu.h` é separado para hosts que não usam GPU não pagarem custo.
- CapyUI **não** chama OpenGL/Vulkan diretamente; apenas produz quads.
- Host CapyOS escolhe backend (Mesa, lavapipe, VirGL, etc.).
