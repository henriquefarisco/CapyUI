# v1.2.0 — GPU path opcional

**Status:** delivered (CapyUI side, 2026-05-21). Backends GPU concretos (Mesa, lavapipe, VirGL) seguem como trabalho do host CapyOS.
**ABI bump:** capy-ui-widget 1.1 → 1.2 (aditivo — novo header opt-in)
**Display-list schema:** permanece 4. O schema bump 4 → 5 do plano original foi deliberadamente **diferido**: o tradutor emite quads em um buffer separado (`capy_gpu_quad[]`), não introduz nova op de DL. Schema só será bumpado quando houver op aditiva no enum `capy_dl_op`.
**Dependência:** v1.1.0 entregue. Não bloqueada.
**Estimativa:** L (CapyUI ✅) + backends GPU (host CapyOS, pendente)
**Owner:** CapyUI

## Objetivo

Translator de display-list para primitivos GPU (quads + scissors + texture refs), sem tornar GPU obrigatório. Host CapyOS decide CPU rasterizer vs. GPU.

## Escopo CapyUI (entregue)

- Novo header `src/widget/capy_dl_gpu.h` (não ifdef-gated; o custo de incluir é desprezível — apenas um struct e uma função). Hosts que não precisam de GPU simplesmente não chamam `capy_dl_to_quads`.
- Novo source `src/widget/capy_dl_gpu.c` adicionado a `SRC_WIDGET` no `Makefile`.
- Struct pública:
  ```c
  struct capy_gpu_quad {
    int32_t x0, y0, x1, y1;
    uint32_t color;
    uint32_t texture_id;     /* 0 = sem textura */
    int32_t uv_x0, uv_y0, uv_x1, uv_y1;
  };
  ```
- API:
  ```c
  int capy_dl_to_quads(const struct capy_display_list *dl,
                       struct capy_gpu_quad *out, uint32_t cap,
                       uint32_t *out_count);
  ```
- Decomposição de ops (determinística, byte-stable, com scissor stack interno `CAPY_DL_GPU_CLIP_STACK_MAX = 16`):
  - `RECT` → 1 quad (rect, color).
  - `BORDER` → 4 quads (top, bottom, left, right strips com `cmd->border_width`).
  - `TEXT` → 1 quad placeholder (texture_id=0, host traduz com seu glyph atlas via `font_id`+`text_offset`+`text_len`).
  - `FOCUS_RING` → 4 quads (mesma forma de BORDER).
  - `IMAGE_REF` → 1 quad com `texture_id = cmd->image_id` e UVs (0,0)-(rect.w, rect.h).
  - `CLIP_PUSH` → stack push interno; intersecta com clip do pai; quads subsequentes são intersectados contra o top da stack. Zero-area clip dropa quads contidos silenciosamente.
  - `CLIP_POP` → stack pop interno.
  - `DIRTY_HINT` / `NONE` → ignorados (DIRTY_HINT é reservado para produtores higher-level; NONE não aparece em DLs bem-formadas).
- Op desconhecida no DL (fora do enum schema 4) → `-1` (fail-closed).
- Stack unbalanced (CLIP_POP sem push, ou push trailing) → `-1`.
- Overflow de `cap` → `-1` com `*out_count` parcial (sem rollback).
- CapyUI **não** chama OpenGL/Vulkan/Metal diretamente; apenas produz quads. Host CapyOS escolhe backend.

## Testes que cobrem o contrato

- `test_gpu_rect_to_single_quad` — RECT (10,20,30,40, color=0xFFAABBCC) → 1 quad com (x0,y0,x1,y1)=(10,20,40,60), color, texture_id=0.
- `test_gpu_border_to_four_quads` — BORDER 50×30 com `border_width=2` → 4 quads (top 50×2, bottom 50×2, left 2×26, right 2×26), mesma cor.
- `test_gpu_clip_intersects_quads` — CLIP_PUSH (10,10,30,30) + RECT (0,0,100,100) → quad (10,10,40,40) intersectado; após CLIP_POP, novo RECT emite unclipped.
- `test_gpu_overflow_returns_minus_one` — cap=1 com 2 RECTs → `-1` com `*out_count=1`; NULL guards.
- `test_gpu_unbalanced_clip_rejected` — CLIP_POP isolado → `-1`; CLIP_PUSH sem pop → `-1`.
- `test_gpu_image_ref_carries_texture_and_uvs` — IMAGE_REF (5,7,32,32, image_id=9001) → quad com texture_id=9001 e UVs (0,0)-(32,32).
- `test_gpu_translator_determinism` — panel+button real → emit → traduzir 2× → quads byte-idênticos.

## Notas

- Header `capy_dl_gpu.h` separado; consumers que não usam GPU simplesmente não linkam `capy_dl_to_quads`.
- CapyUI **não** chama OpenGL/Vulkan/Metal diretamente; apenas produz quads.
- Host CapyOS escolhe backend (Mesa, lavapipe, VirGL, etc.).
- Schema do display-list permanece 4 (não bumpado para 5 como o plano original sugeria). O tradutor opera sobre o schema 4 atual sem novas ops; um bump para 5 ficará reservado para quando uma op realmente nova chegar ao `capy_dl_op`.
