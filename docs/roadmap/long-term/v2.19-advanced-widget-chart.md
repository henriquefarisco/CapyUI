## Contexto

Sexta fatia da trilha de **estado dos advanced widgets** (date 2.14, color 2.15,
table 2.16, autocomplete 2.17, tree 2.18). Como a table/autocomplete guarda um
**array caller-owned** (zero-alloc); diferente de todas as anteriores, adiciona
uma **redução numérica** sobre os dados: `capy_widget_chart_range` faz um scan
single-pass do min/max assinado. Mantém os invariantes: zero-alloc, zero-float,
total, determinístico, fail-closed por tipo.

## Escopo CapyUI (entregue)

### 1. Campos de cauda

```c
const int32_t *chart_values; /* caller-owned; NULL = sem dados */
uint16_t       chart_count;  /* nº de pontos; 0 = sem dados */
```

Appended ao tail de `struct capy_widget`, válidos só quando
`type == CAPY_WIDGET_CHART`. Pontos são `int32_t` assinados (suportam valores
negativos). Array caller-owned (mesmo idioma da table/autocomplete) que deve
sobreviver ao uso pelo widget. NULL/0 (default zero-init) = "sem dados".
`sizeof(struct capy_widget)` cresce de 864 (2.18) para 880 bytes.

### 2. APIs (5: total, determinístico, zero-alloc, zero-float, fail-closed)

- `int capy_widget_set_chart_data(w, values, count)` — armazena o dataset.
  `count == 0` limpa (values ignorado). `count > 0` exige `values != NULL`.
  `0`; `-1` em NULL / `type != CHART` / (`count > 0 && values == NULL`).
  Fail-closed (dataset intacto em falha).
- `int capy_widget_clear_chart_data(w)` — volta a NULL/0.
- `int capy_widget_chart_count(w)` — nº de pontos (0 quando nenhum); `-1` em NULL/tipo.
- `int capy_widget_chart_value(w, index, out_value)` — escreve `values[index]`
  em `*out_value`; `-1` em NULL / tipo / `out_value==NULL` / sem dados /
  `index >= count` (bounds-check).
- `int capy_widget_chart_range(w, out_min, out_max)` — scan single-pass do
  min/max assinado. Retorna `1` com os limites escritos quando há dados, `0`
  (e `*out_min = *out_max = 0`) quando vazio, `-1` em NULL / tipo / `out_min`
  NULL / `out_max` NULL.

DL schema permanece 7. `capy_widget_serialize` (1.10) **não** serializa o
dataset (ponteiro caller-owned).

## Testes obrigatórios (entregues — 8)

1. `chart_set_and_query` — set [10,20,30], count 3, value(0)/value(2).
2. `chart_value_out_of_range` — sem dados → -1; `index == count` e além → -1.
3. `chart_set_zero_count_clears` — `count==0` limpa mesmo com ponteiro passado.
4. `chart_set_null_values_rejected` — `set(NULL, count>0)` → -1, dataset intacto.
5. `chart_clear` — set → clear → count 0, value → -1, range → 0 (out zerados).
6. `chart_range_minmax` — [5,-3,42,17] → min -3, max 42 (assinado); single → min==max.
7. `chart_wrong_widget_type_rejected` — as 5 APIs em BUTTON → -1.
8. `chart_null_guards` — NULL widget; NULL out_value; NULL out_min/out_max.
