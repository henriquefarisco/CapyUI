# v1.4.0 — Gesture engine

**Status:** delivered (CapyUI side, 2026-05-21) com escopo single-touch. PINCH_IN/OUT e ROTATE_CW/CCW ficam **reservados no enum** mas não emitidos pelo recognizer 1.4 (multi-touch fica para fatia futura aditiva).
**ABI bump:** capy-ui-widget 1.3 → 1.4 (aditivo)
**Display-list schema:** permanece 4 (gesture engine não emite no display-list; o schema 5 do cabeçalho original era placeholder)
**Dependência:** v1.3.0 entregue + v0.10.0 (touch events)
**Estimativa:** L (CapyUI ✅)
**Owner:** CapyUI

## Objetivo

Reconhecedor de gestos determinístico: tap, double-tap, long-press, swipe (4 direções), pinch, rotate, drag. Mesma entrada sintética → mesma saída.

## Escopo CapyUI

- Enum:
  ```c
  enum capy_gesture_kind {
    CAPY_GESTURE_NONE = 0,
    CAPY_GESTURE_TAP,
    CAPY_GESTURE_DOUBLE_TAP,
    CAPY_GESTURE_LONG_PRESS,
    CAPY_GESTURE_SWIPE_LEFT,
    CAPY_GESTURE_SWIPE_RIGHT,
    CAPY_GESTURE_SWIPE_UP,
    CAPY_GESTURE_SWIPE_DOWN,
    CAPY_GESTURE_PINCH_IN,
    CAPY_GESTURE_PINCH_OUT,
    CAPY_GESTURE_ROTATE_CW,
    CAPY_GESTURE_ROTATE_CCW,
    CAPY_GESTURE_DRAG
  };
  ```
- Struct:
  ```c
  struct capy_gesture {
    enum capy_gesture_kind kind;
    struct capy_ui_point start;
    struct capy_ui_point end;
    int32_t magnitude;        /* pixels para swipe, % para pinch, graus*256 para rotate */
    uint32_t duration_ms;
  };
  ```
- Recognizer (struct concreta, não opaca, em linha com o resto do projeto; estado ≈ 80 bytes, todo embutido):
  ```c
  struct capy_gesture_recognizer {
    /* thresholds caller-tunáveis (defaults via _init) */
    uint32_t tap_max_duration_ms;     /* 200 */
    uint32_t tap_max_distance_px;     /* 10  */
    uint32_t double_tap_max_gap_ms;   /* 300 */
    uint32_t long_press_min_ms;       /* 500 */
    uint32_t swipe_min_distance_px;   /* 50  */
    /* estado interno (single-touch) */
    uint8_t  active;
    uint8_t  long_press_emitted;
    uint8_t  drag_emitted;
    uint8_t  has_pending;
    uint8_t  has_last_tap;
    uint8_t  reserved[3];
    uint32_t touch_id;
    struct capy_ui_point start_pos;
    struct capy_ui_point last_pos;
    uint32_t start_tick;
    uint32_t last_tap_end_tick;
    struct capy_ui_point last_tap_pos;
    struct capy_gesture  pending;     /* depth 1 */
  };

  void capy_gesture_recognizer_init(struct capy_gesture_recognizer *r);
  int  capy_gesture_feed(struct capy_gesture_recognizer *r,
                         const struct capy_widget_event *ev, uint32_t now);
  int  capy_gesture_tick(struct capy_gesture_recognizer *r, uint32_t now);
  int  capy_gesture_poll(struct capy_gesture_recognizer *r,
                         struct capy_gesture *out);
  ```
- `capy_gesture_feed` ganhou `uint32_t now` em relação ao cabeçalho original do plano — `struct capy_widget_event` não carrega timestamp (since 0.0), então o host precisa fornecer o tick monótonico aqui (mesma convenção de `capy_anim_sample`). `capy_gesture_tick` foi adicionado para permitir emit de LONG_PRESS sem precisar de novo evento.
- Constantes documentadas: thresholds default em `_init` (tap 200 ms / 10 px; double-tap gap 300 ms; long-press 500 ms; swipe 50 px).

## Testes que cobrem o contrato

- `test_gesture_tap_emits_once` — begin→end em 80 ms sem movimento emite TAP; segundo poll retorna 0 (queue vazia).
- `test_gesture_double_tap_within_gap` — dois taps separados por 150 ms emitem TAP + DOUBLE_TAP (em ordem).
- `test_gesture_long_press_via_tick` — begin sem release; após `tick(now+600 ms)` emite LONG_PRESS; release posterior não emite TAP.
- `test_gesture_swipe_four_directions` — begin em (100,100) e end em (200,100) emite SWIPE_RIGHT; analogamente esquerda/cima/baixo.
- `test_gesture_drag_emitted_at_threshold_cross` — begin + move pequeno (não emite) + move passando swipe_min_distance_px emite DRAG; em seguida o release não emite swipe (já era drag).
- `test_gesture_determinism` — mesma sequência (eventos + now ticks) executada 2× produz os mesmos gestos com os mesmos campos.
- `test_gesture_ignores_non_touch_and_extra_touch_ids` — KEY/POINTER/WHEEL/GAMEPAD events retornam 0 sem efeito; um segundo TOUCH_BEGIN com `touch_id` diferente enquanto já ACTIVE é ignorado (recognizer não corrompe estado; multi-touch deferido).

PINCH/ROTATE não têm teste em 1.4 porque são multi-touch e ficam para fatia futura.

## Notas

- Determinístico: thresholds documentados; mesma sequência de `(event, now)` produz a mesma saída em todos os builds.
- Memory: recognizer é struct concreta embutida no caller — zero malloc. Buffer extra não é necessário neste minor.
- Host fornece touch events via v0.10 plumbing (`CAPY_WIDGET_EVENT_TOUCH_BEGIN/MOVE/END` + `capy_touch_payload`). O recognizer usa `payload->pos` (não `event->x/y`) para imunidade contra hosts que esqueçam de copiar o pos.
- Multi-touch (PINCH_IN/OUT, ROTATE_CW/CCW) **não emite em 1.4**. Os valores ficam no enum como ABI reservada (aditivo) para uma fatia futura que adicionar tracking de múltiplos `touch_id`s. Hosts podem comparar `g.kind` contra esses valores em código de chamada sem quebrar quando o recognizer começar a emití-los.
