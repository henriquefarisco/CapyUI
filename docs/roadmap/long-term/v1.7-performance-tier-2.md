## Objetivo

Display-list compression (delta encoding), cache de measure com versionamento, allocator slab para widgets pequenos. Alvo: 100k widgets em 16ms.

## Escopo CapyUI

- Display-list compression:
  - Frame N+1 codifica diff vs. frame N quando possível.
  - API `capy_display_list_compress(prev, next, out_buf, &out_len)`.
  - API `capy_display_list_decompress(buf, len, &out_dl)`.
  - Formato canônico documentado em `contracts/display-list.md`.
- Cache de measure:
  - `capy_widget_context` ganha tabela hash de subarvore→measure result.
  - Invalidação por `layout_version` monotônico.
- Slab allocator:
  ```c
  struct capy_slab {
    void *buffer;
    uint32_t size;
    uint32_t element_size;
    uint32_t high_water;
    void *free_list;
  };

  void capy_slab_init(struct capy_slab *s, void *buf, uint32_t size, uint32_t elem_size);
  void *capy_slab_alloc(struct capy_slab *s);
  void capy_slab_free(struct capy_slab *s, void *ptr);
  ```

## Testes que cobrem o contrato

- `test_slab_init_partitions_buffer` — init seta buffer/size/element_size, zera high_water e free_list.
- `test_slab_alloc_returns_distinct_pointers_until_full` — 4 alloc-out-of-4 retornam ponteiros distintos; 5ª alloc retorna NULL; `high_water == 4`.
- `test_slab_free_returns_to_lifo_pool` — free(a), free(b) seguidos de alloc, alloc retornam b, a (LIFO).
- `test_slab_degenerate_init_returns_null_on_alloc` — element_size < sizeof(void*), NULL buffer, size==0: `size` clampa para 0 e `_alloc` retorna NULL.
- `test_slab_null_guards` — init(NULL), alloc(NULL), free(NULL,...) e free(...,NULL) são no-ops.
- `test_slab_alloc_pointer_alignment` — ponteiros retornados têm alinhamento `sizeof(void*)`; slots consecutivos estão exatamente `element_size` bytes à frente.
- `test_slab_determinism_same_addresses` — dois slabs sobre buffers gemelos consumindo a mesma sequência `(alloc, alloc, free, alloc, alloc, alloc, alloc)` produzem os mesmos índices relativos de slot.

**Não testado em 1.7** (deferido):
- Compress/decompress round-trip do display-list — `capy_widget_diff` (since 0.8) e `capy_display_list_diff_incremental` (since 1.1) já cobrem o caso "transmitir só o que mudou". Compress format ficará para fatia futura aditiva.
- Hash measure-cache de subtree — `layout_dirty` + `layout_version` (since 0.15) já cobrem invalidação coarse. Hash table por subtree fica para fatia futura.
- Benchmark 100k widgets em 16 ms — gate externo (não executável neste workspace por política local). Recomenda-se rodar no CI CapyOS.
- Regressão vs baseline v0.15 — mesma justificativa; gate externo.

## Notas

- **Slab** é zero-alloc (caller fornece o buffer) e determinístico LIFO. Compatibilidade lado-a-lado com `capy_widget_pool` (since 0.15): pool para arenas monotonicamente crescentes, slab para tight loops com free+reuse frequentes.
- A free-list mora inline nos slots liberados — portanto `element_size >= sizeof(void *)`. Init com tamanho menor clampa `size = 0` e todos os `_alloc` subsequentes retornam NULL (fail-closed).
- Sem validação de range em `_free`: caller é responsável por passar apenas ponteiros previamente retornados pelo mesmo slab. Mantém `free` O(1).
- Compression e hash cache adiados para fatia futura aditiva. Métricas registradas em `tracking/METRICS.md` por release — quando disponêveis (workspace não executa benchmarks).
