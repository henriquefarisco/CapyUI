## Objetivo

Display-list compression (delta encoding), cache de measure com versionamento, allocator slab para widgets pequenos. Alvo: 100k widgets em 16ms.

## Escopo CapyUI

- Display-list compression:
  - Frame N+1 codifica diff vs. frame N quando possível.
  - API `capy_display_list_compress(prev, next, out_buf, &out_len)`.
  - API `capy_display_list_decompress(buf, len, &out_dl)`.
  - Formato canônico documentado em `contracts/display-list.md`.
- Cache de measure:
  - `capy_widget_context` ganha tabela hash de subarvore→measure result.
  - Invalidação por `layout_version` monotônico.
- Slab allocator:
  ```c
  struct capy_slab {
    void *buffer;
    uint32_t size;
    uint32_t element_size;
    uint32_t high_water;
    void *free_list;
  };

  void capy_slab_init(struct capy_slab *s, void *buf, uint32_t size, uint32_t elem_size);
  void *capy_slab_alloc(struct capy_slab *s);
  void capy_slab_free(struct capy_slab *s, void *ptr);
  ```

## Testes obrigatórios

1. **Compress/decompress round-trip** byte-a-byte.
2. **Cache de measure:** segunda chamada com mesma árvore é O(1).
3. **Slab alloc/free** sem fragmentação.
4. **Benchmark: 100k widgets** measure+arrange+emit em < 16ms (alvo).
5. **Sem regressão > 10%** vs. baseline v0.15.

## Notas

- Compression é opcional; compositor pode escolher full vs. diff.
- Slab é uma das estratégias do allocator; caller continua escolhendo.
- Métricas registradas em `tracking/METRICS.md` por release.
