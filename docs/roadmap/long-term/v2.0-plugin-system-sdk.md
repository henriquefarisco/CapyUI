# v2.0.0 — Plugin system + SDK público

**Status:** delivered (CapyUI side, 2026-05-21). **Major bump 1.10 → 2.0** com escopo conservador: superfície de plugin (`capy_plugin_descriptor`, registry, register/unregister APIs), op `CAPY_DL_PLUGIN_OP` aditiva (DL schema 6→7), seed do diretório `sdk/` (README + plugin-template.c). **Nenhum símbolo do 1.x foi removido** — a quebra controlada permitida por SemVer 2.0 fica armada para minors 2.x sob a `deprecation-policy.md`. LTS de 1.x permanece pelo período mínimo de 12 meses.
**ABI bump:** capy-ui-widget 1.10 → **2.0 (major)**
**Display-list schema:** **6 → 7** (`CAPY_DL_PLUGIN_OP`). Terceiro bump real desde 0.9 (4→5 em 1.5, 5→6 em 1.9, 6→7 em 2.0).
**Dependência:** v1.10.0 entregue + v1.0 LTS ativa
**Estimativa:** XL (CapyUI ✅)
**Owner:** CapyUI

## Objetivo

Plataforma extensível. Widgets de terceiros via ABI estável `capy-ui-plugin`. SDK público com headers, samples e tutoriais.

## Escopo CapyUI

- Novo ABI `capy-ui-plugin` separado do `capy-ui-widget`.
- Header `src/widget/capy_plugin.h`:
  ```c
  struct capy_plugin_descriptor {
    const char *id;                /* "com.example.myplugin" */
    const char *version;           /* "1.0.0" */
    uint32_t capy_ui_abi_required; /* ex. 0x00020000 = 2.0 */

    int (*init)(struct capy_plugin_context *ctx);
    void (*destroy)(struct capy_plugin_context *ctx);
    int (*on_event)(struct capy_plugin_context *ctx,
                    struct capy_widget *target,
                    const struct capy_widget_event *ev);
    int (*emit)(struct capy_plugin_context *ctx,
                struct capy_widget *target,
                struct capy_display_list *dl);
  };

  int capy_plugin_register(struct capy_widget_context *ctx,
                           const struct capy_plugin_descriptor *desc);
  ```
- Plugin context isola allocator e estado por plugin.
- Sandbox em runtime:
  - Timeout por callback (host fornece).
  - Allocator dedicado por plugin.
  - Sem acesso a `capy_widget_context` global do host.
- Display-list schema 8 ganha `CAPY_DL_PLUGIN_OP` aditivo:
  - Payload genérico (32 bytes).
  - Compositor delega para plugin renderer registrado.

### Quebras controladas em 2.0

- Símbolos deprecados em v1.x são removidos.
- Algumas APIs antigas ganham wrappers de compatibilidade (`_v1`) por LTS de 12 meses.
- Headers renomeiam internamente `_internal_*` para deixar API pública mais limpa.

### SDK público

- Diretório `sdk/` no repo:
  - Headers públicos curados.
  - Exemplos: widget custom, plugin de tema, plugin de animação.
  - Tutoriais markdown.
  - Build script de plugin (`make plugin-template`).
- Documentação em `docs/sdk/`.

## Testes que cobrem o contrato

- `test_plugin_version_tag_macro` — `CAPYUI_API_VERSION_TAG == 0x00020000` (major=2, minor=0, patch=0).
- `test_plugin_register_valid_descriptor` — descritor válido aceita; duplicate de mesmo ponteiro rejeitado.
- `test_plugin_rejects_abi_mismatch` — `capy_ui_abi_required > CAPYUI_API_VERSION_TAG` (e.g. 0x00090000) rejeitado; valores `==` e `<` aceitos.
- `test_plugin_capacity_overflow_rejected` — `CAPY_MAX_PLUGINS` (8) registrados; o 9º retorna -1 sem mutação.
- `test_plugin_unregister_all_calls_destroy` — `destroy` chamado uma vez por descritor com callback presente; idempotente em chamadas subsequentes.
- `test_plugin_null_guards` — `register(NULL, *)`, `register(*, NULL)` retornam `-1`; `unregister_all(NULL)` é no-op.
- `test_plugin_op_in_dl_skipped_by_gpu_translator` — `capy_dl_to_quads` skipa `CAPY_DL_PLUGIN_OP` (metadata para plugin renderer); processa RECT vizinho normalmente.
- `test_plugin_dl_schema_bumped_to_7` — macro + display-list runtime reportam schema 7; `CAPY_DL_PLUGIN_OP > CAPY_DL_TRANSFORM_POP`.

**Não testado em 2.0** (deferido):
- Plugin crash isolation (signal handling) — widget core nunca invoca callbacks; host implementa watchdog.
- Timeout enforcement em runtime — `timeout_microseconds` é hint; host enforça externamente.
- Allocator isolation auditing — descritor carrega `capy_widget_allocator` separado; core não mistura mas também não usa.
- Compatibility shims `_v1` — nenhuma API foi renomeada/removida em 2.0; política fica ARMADA para minors 2.x.
- Plugin-side `on_event`/`emit` invocation — host dispatcher loop fica fora do widget core.

## Notas

- **Quebra controlada SIM no SemVer, NO nas APIs.** A bump 1.x→2.0 é obrigatória pelo SemVer porque adicionamos uma op nova (`CAPY_DL_PLUGIN_OP`) que pode aparecer no display-list e quebrar consumers ingenuos pre-schema-7, e porque introduzimos um novo ABI (`capy-ui-widget` 2.0) que cumpre os pré-requisitos para futuras remoções sob a `deprecation-policy.md`. Mas **nenhuma função existente foi removida ou renomeada em 2.0.0**. Compatível em código-fonte com qualquer host 1.10.0 (modulo as 3 asserts de schema literal). Mítico: o major-bump não quebra código, abre porta para quebrar em 2.x.
- 1.x permanece em LTS por **≥12 meses pós-2.0** conforme `docs/roadmap/contracts/deprecation-policy.md`.
- SDK lança junto com 2.0 em `sdk/`: `README.md` curador + `plugin-template.c` compilable.
- Plugin marketplace pode emergir no CapyOS (não escopo do CapyUI core).
- O widget core **não invoca callbacks** de plugin. Host dispatcher itera o registro e aplica seu próprio sandbox (timeout, signal handling). Mantém core float-free, determinístico e crash-isolado por construção.
- `sizeof(capy_widget_context)` cresce com o registro de 8 ponteiros + count. Hosts que stack-allocate context precisam recompilar.
- Macros bumpadas para 2/0/0 + novo `CAPYUI_API_VERSION_TAG = 0x00020000`.
