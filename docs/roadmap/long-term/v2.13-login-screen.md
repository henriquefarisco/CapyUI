# v2.13.0 — Login screen with user avatars

**Status:** delivered (2026-05-21) — **fecha a fase 2.x (14/14)**.
**ABI bump:** capy-ui-widget 2.12 → 2.13 (aditivo: taxonomy + layout chooser) + capy-ui-desktop-session 1 (aditivo, sem bump)
**Display-list schema:** 7 (sem mudança)
**Dependência:** v2.7 user management, v2.11 icons & thumbnails, v2.12 wallpape
**Estimativa:** M

## Escopo entregue (widget core)

Plumbing minimalista — taxonomia estável + decisão determinística de layout. Widget core não autentica (v2.7), não compõe background (v2.12), não decode avatares (v2.11). UX completa vive no desktop-session (`src/desktop/login.c`).

### Macros (12)

- 4 layout: `CAPY_LOGIN_LAYOUT_SINGLE` (=0), `_GRID` (=1), `_LIST` (=2), `_COUNT` (=3).
- 2 thresholds: `CAPY_LOGIN_GRID_THRESHOLD` (=2), `CAPY_LOGIN_LIST_THRESHOLD` (=7).
- 7 power: `CAPY_LOGIN_POWER_NONE/SHUTDOWN/REBOOT/SLEEP/LOCK/LOGOUT/COUNT` (= 0..6).

### 1 API determinística

- `uint8_t capy_login_choose_layout(uint32_t user_count)` — `0|1 → SINGLE`; `2..6 → GRID`; `≥7 → LIST`. Zero-alloc, zero-float, total-function (sem -1).

## Escopo desktop-session

Login screen widget tree em `src/desktop/login.c` consome:
- `capy_user_directory` (v2.7) para enumerar contas + autenticar.
- `capy_login_choose_layout` (v2.13) para escolher SINGLE/GRID/LIST.
- `capy_icon_resolve` (v2.11) para avatares via `user.avatar_image_id`.
- `capy_wallpaper_*` (v2.12) para background.
- `capy_login_power_*` (v2.13) para botões shutdown/reboot/sleep/lock.

Mesmas APIs alimentam o **lock screen** (Super+L) e o **user switcher** (taskbar) — uma única math, três surfaces.

## Testes entregues (6)

1. `login_layout_zero_or_one` — 0 e 1 → SINGLE.
2. `login_layout_grid_range` — 2..6 → GRID.
3. `login_layout_list_threshold` — 6→GRID, 7→LIST, UINT32_MAX→LIST.
4. `login_layout_deterministic` — 32× chamadas com mesmo input → mesma saída.
5. `login_layout_taxonomy` — SINGLE/GRID/LIST/COUNT = 0/1/2/3 + thresholds estáveis.
6. `login_power_taxonomy` — NONE/SHUTDOWN/REBOOT/SLEEP/LOCK/LOGOUT/COUNT = 0..6.

## Decisões técnicas

- **Total function.** `capy_login_choose_layout` nunca falha — todos os `uint32_t` mapeiam para um layout válido.
- **Threshold em 7.** Acima de 6 avatares, grid 2×3 começa a ficar denso; lista + search é mais escalável.
- **Single layout cobre 0 e 1.** Zero usuários → first-boot create-root prompt; um usuário → form direto.
- **Power taxonomy estável.** Hosts podem ocultar subconjunto (kiosk geralmente esconde SHUTDOWN/REBOOT) sem alterar a enumeração.

## Deferido para minors futuros

- **Biometric / SSO challenge UI.** Novo `capy_login_challenge` enum + per-method affordances.
- **PIN keypad widget.** Composite desktop-session.
- **Locked-badge animation.** Visual do bit `is_locked` (v2.7).
- **Resume-from-sleep wake screen.** Reusa `capy_login_choose_layout`.

## Objetivo

Tela de login redesenhada: lista de usuários com avatares grandes, troca rápida entre contas, suporte a guest, power options visíveis, layout adaptativo (1 usuário → form direto; 2-6 → grid; 7+ → lista com search).

## Escopo desktop-session

- Novo app `src/apps/login.c` (ou refatoração do existente).
- Layout determinado por `user_directory` count (vindo de v2.7):
  - 1 user: avatar grande centrado + username + password field.
  - 2–6 users: grid 3 colunas, avatares 128×128, click expande para password.
  - 7+ users: lista vertical com search box no topo, avatares 64×64.
- Cada user tile mostra: avatar (v2.11 resolve), display_name, "Last login: 2 days ago" (opcional via `last_login_ms`).
- Estados visuais:
  - User normal: BG_RAISED + FG_PRIMARY.
  - User locked: badge cadeado + DISABLED token.
  - Selecionado: FOCUS_RING + ACCENT background sutil.
- Power options (canto inferior direito): Shutdown, Restart, Sleep, Switch user, Guest session.
- Acessibilidade: Tab cicla users; Enter expande; Esc colapsa de volta para lista.
- Wallpaper de login independente do desktop (v2.12 sem `per_monitor`, com `image_id` global do sistema).
- Indicadores de hora, data, bateria, rede no top bar.

## Escopo CapyUI core

- Nenhum bump. Compõe widgets existentes (LIST, TEXTBOX, IMAGE, BUTTON) com tokens + DPI + icons.
