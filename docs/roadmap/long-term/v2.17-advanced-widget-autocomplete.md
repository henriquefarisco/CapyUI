# v2.17.0 — Advanced widget state: autocomplete suggestions

**Status:** delivered (commit-side; gates externos pendentes como toda tag)
**ABI bump:** capy-ui-widget 2.16 → **2.17 (aditivo)**
**Display-list schema:** permanece **7**
**Dependência:** v2.1 (advanced widget enums) + v2.14/v2.15/v2.16 (padrão de estado estabelecido)
**Estimativa:** S
**Owner:** —

## Contexto

Quarta fatia da trilha de **estado dos advanced widgets** (date picker 2.14,
color picker 2.15, table columns 2.16). Como a table, guarda um **modelo
caller-owned** (zero-alloc); diferente de todas as anteriores, adiciona um
**cursor de seleção mutável** com semântica de _clamp_ / fail-closed. Mantém os
invariantes: zero-alloc (CapyUI nunca copia/aloca/libera o array de strings),
zero-float, total, determinístico.

## Escopo CapyUI (entregue)

### 1. Campos de cauda

```c
const char *const *autocomplete_items; /* caller-owned; NULL = sem lista */
uint16_t           autocomplete_count; /* nº de sugestões; 0 = sem lista */
int32_t            autocomplete_selected; /* índice destacado, ou -1 = nenhum */
```

Appended ao tail de `struct capy_widget`, válidos só quando
`type == CAPY_WIDGET_AUTOCOMPLETE`. A lista usa exatamente o idioma caller-owned
nunca-copiado de `drop_accepted_types` (1.6) e da candidate list do IME (1.8).
`autocomplete_selected` é **clampado contra a contagem viva na leitura**, então
uma seleção obsoleta nunca aponta além da lista (e o default zero-init de um
widget novo é mascarado para "nenhum"). `sizeof(struct capy_widget)` cresce de
840 (2.16) para 856 bytes.

### 2. APIs (6: total, determinístico, zero-alloc, zero-float, fail-closed)

- `int capy_widget_set_autocomplete(w, items, count)` — armazena a lista e
  **reseta a seleção para -1**. `count == 0` limpa (items ignorado). `count > 0`
  exige `items != NULL`. `0`; `-1` em NULL `w` / `type != AUTOCOMPLETE` /
  (`count > 0 && items == NULL`). Fail-closed (modelo intacto em falha).
- `int capy_widget_clear_autocomplete(w)` — volta a NULL/0 e seleção -1.
- `int capy_widget_autocomplete_count(w)` — nº de sugestões (0 quando nenhuma);
  `-1` em NULL / tipo errado.
- `int capy_widget_autocomplete_item(w, index, out_item)` — escreve
  `items[index]` em `*out_item`; `-1` em NULL / tipo / `out_item==NULL` / sem
  lista / `index >= count` (bounds-check).
- `int capy_widget_set_autocomplete_selected(w, index)` — `index == -1` limpa a
  seleção; `0 <= index < count` seleciona. `-1` em NULL / tipo / `index < -1` /
  `index >= count` (fail-closed).
- `int capy_widget_get_autocomplete_selected(w, out_index)` — escreve a seleção
  viva (clampada; -1 quando nenhuma) em `*out_index`. Retorna `1` quando há
  seleção, `0` quando nenhuma, `-1` em NULL / tipo / `out_index==NULL`.

DL schema permanece 7. `capy_widget_serialize` (1.10) **não** serializa a lista
(strings caller-owned) nem o índice de seleção.

## Testes obrigatórios (entregues — 8)

1. `autocomplete_set_and_query` — set 3 itens, count 3, item(0)/item(2) por ponteiro.
2. `autocomplete_item_out_of_range` — sem lista → -1; `index == count` e além → -1.
3. `autocomplete_set_zero_count_clears` — `count==0` limpa mesmo com ponteiro passado.
4. `autocomplete_set_null_items_rejected` — `set(NULL, count>0)` → -1, modelo intacto.
5. `autocomplete_clear` — set + select → clear → count 0, item → -1, seleção -1.
6. `autocomplete_selection_clamp` — default nenhum; select válido; -1 limpa;
   out-of-range (`>= count`, `< -1`) rejeitado; nova lista mais curta reseta a seleção.
7. `autocomplete_wrong_widget_type_rejected` — as 6 APIs em BUTTON → -1.
8. `autocomplete_null_guards` — NULL widget; NULL out_item; NULL out_index.

Total da suíte: **297 → 305** (guard de contagem pinado em `make version-check`).

## Compatibilidade

- 100% aditivo. Macros bumpadas para `2/17/0`; `CAPYUI_API_VERSION_TAG = 0x00021100`.

  não muda. `sizeof(capy_widget)` 840 → 856 (ponteiro + uint16 + int32, com padding).
