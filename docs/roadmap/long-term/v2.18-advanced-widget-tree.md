## Contexto

Quinta fatia da trilha de **estado dos advanced widgets** (date picker 2.14,
color picker 2.15, table columns 2.16, autocomplete 2.17). É a primeira cujo
lado de leitura **caminha a hierarquia de widgets**: a visibilidade de uma linha
da árvore é derivada do estado de fold dos seus ancestrais `TREE`. Mantém os
invariantes: zero-alloc, zero-float, total, determinístico, fail-closed por tipo.

## Escopo CapyUI (entregue)

### 1. Campos de cauda

```c
uint8_t  tree_collapsed; /* 1 = filhos escondidos; default 0 = expandido */
uint16_t tree_depth;     /* nível de indentação; default 0 = raiz */
```

Appended ao tail de `struct capy_widget`, válidos só quando
`type == CAPY_WIDGET_TREE`. O default zero-init (`tree_collapsed == 0`) significa
**expandido**, então um nó novo mostra os filhos. `tree_depth` é um nível de
indentação setado pelo caller (0 = raiz), útil para modelos de árvore
achatados/virtualizados onde o aninhamento visual não precisa espelhar a árvore
de widgets. `sizeof(struct capy_widget)` cresce de 856 (2.17) para 864 bytes.

### 2. APIs (5: total, determinístico, zero-alloc, zero-float, fail-closed)

- `int capy_widget_set_tree_collapsed(w, collapsed)` — seta o flag de fold
  (qualquer `collapsed` não-zero armazena 1). `0`; `-1` em NULL / `type != TREE`.
- `int capy_widget_tree_is_collapsed(w)` — `1` collapsed, `0` expandido, `-1` em
  NULL / tipo errado. Nó novo é expandido (0).
- `int capy_widget_set_tree_depth(w, depth)` — seta o nível de indentação.
  `0`; `-1` em NULL / tipo errado.
- `int capy_widget_tree_depth(w)` — nível de indentação (0 na raiz), `-1` em
  NULL / tipo errado.
- `int capy_widget_tree_row_visible(w)` — **caminha a cadeia de pais**: retorna
  `0` quando qualquer ancestral que é um nó `TREE` está collapsed (o próprio fold
  do nó NÃO o esconde), `1` quando nenhum ancestral `TREE` collapsed é
  encontrado, `-1` em NULL / `type != TREE`. Determinístico; ancestrais não-`TREE`
  são pulados.

DL schema permanece 7. `capy_widget_serialize` (1.10) **não** serializa o estado
de fold nem a profundidade (estado de UI efêmero).

## Testes obrigatórios (entregues — 8)

1. `tree_collapsed_default_expanded` — nó novo: is_collapsed 0, depth 0.
2. `tree_set_collapsed_toggle` — set 1/0; qualquer não-zero normaliza p/ 1.
3. `tree_depth_set_and_get` — default 0; set 3 → 3; set 0 → 0.
4. `tree_row_visible_no_ancestors` — nó sem pai sempre visível (mesmo collapsed).
5. `tree_row_visible_collapsed_ancestor` — pai collapsed esconde o filho; expandir revela.
6. `tree_row_visible_nested` — avô collapsed esconde tudo abaixo; pai collapsed
   esconde o filho mas não a si mesmo (semântica "só ancestrais").
7. `tree_wrong_widget_type_rejected` — as 5 APIs em BUTTON → -1.
8. `tree_null_guards` — NULL widget nas 5 APIs → -1.

Total da suíte: **305 → 313** (guard de contagem pinado em `make version-check`).

## Compatibilidade

- 100% aditivo. Macros bumpadas para `2/18/0`; `CAPYUI_API_VERSION_TAG = 0x00021200`.
- Estado de fold/depth é estado de UI efêmero: `capy_widget_serialize` não muda.
  `sizeof(capy_widget)` 856 → 864 (uint8 + uint16, com padding).
