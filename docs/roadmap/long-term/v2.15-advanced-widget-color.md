## Contexto

Segunda fatia da trilha de **estado dos advanced widgets** (a primeira foi
`v2.14` date picker). Levanta o `COLOR_PICKER` do `user_data` para um campo de
cauda de primeira classe, seguindo exatamente o mesmo formato aditivo:
total / determinístico / zero-alloc / zero-float / fail-closed por tipo.

## Escopo CapyUI (entregue)

### 1. Campos de cauda

```c
uint32_t picker_color;     /* 0xAARRGGBB */
uint8_t  picker_color_set; /* 0 = unset (default), 1 = set */
```

Appended ao tail de `struct capy_widget`, válidos só quando
`type == CAPY_WIDGET_COLOR_PICKER`. Como **todo** valor de 32 bits é uma cor
válida, não há sentinela natural de "unset" — daí o flag `picker_color_set`
(contraste com o date picker, cuja sentinela é `year/month/day == 0`).
`sizeof(struct capy_widget)` cresce de 816 para 824 bytes (a cor reaproveita
o padding de cauda que existia após `date_value`; +1 byte do flag + padding).

### 2. APIs (total, determinístico, zero-alloc, zero-float)

- `uint32_t capy_color_pack(uint8_t r, uint8_t g, uint8_t b, uint8_t a)` —
  empacota canais de 8 bits em `0xAARRGGBB` (mesma ordem de
  `capy_widget_style.*_color` e dos theme tokens). Cada canal é convertido
  para `uint32_t` **antes** do shift, evitando UB de shift com sinal no byte
  de alpha (cf. limpeza strict-C11 da 1.3). O inverso é o idioma já usado no
  core: `(argb >> 16) & 0xFFu` (R) / `>> 8` (G) / `& 0xFFu` (B) / `>> 24` (A).
- `int capy_widget_set_color(w, argb)` — armazena `argb` e marca como set.
  `0` em sucesso; `-1` em NULL `w` ou `w->type != CAPY_WIDGET_COLOR_PICKER`.
  Sem validação de valor (toda cor é válida).
- `int capy_widget_clear_color(w)` — volta ao unset (cor 0, flag 0). `0`;
  `-1` em NULL / tipo errado.
- `int capy_widget_get_color(w, out)` — copia a cor em `*out` (0 quando
  unset). Retorna `1` set, `0` unset, `-1` em NULL `w` / tipo errado /
  `out == NULL`.

DL schema permanece 7: o COLOR_PICKER continua emitindo pelo pipeline
genérico; esta fatia só adiciona estado.

## Testes obrigatórios (entregues — 8)

1. `color_pack_channel_order` — `pack(0x12,0x34,0x56,0x78) == 0x78123456` + extração por canal.
2. `color_pack_extremes` — branco/preto/opaque-black (cobre o byte de alpha sem UB).
3. `color_set_and_get` — set + get retorna 1 e o valor.
4. `color_unset_get_returns_zero` — widget novo retorna 0, escreve 0.
5. `color_clear_resets` — set → clear → get retorna 0.
6. `color_wrong_widget_type_rejected` — set/clear/get em BUTTON → -1.
7. `color_null_guards` — NULL widget e NULL out → -1.
8. `color_determinism` — mesma escrita → mesma leitura repetida.

Total da suíte: **281 → 289** (guard de contagem pinado em `make version-check`).

## Compatibilidade

- 100% aditivo. Macros bumpadas para `2/15/0`; `CAPYUI_API_VERSION_TAG = 0x00020F00`.
- `capy_widget_serialize` (1.10) **não** serializa `picker_color`/`picker_color_set`
  (continuam na lista de campos deferidos); o formato binário não muda.

## Deferido para fatias seguintes

- Demais famílias advanced-widget (TABLE columns, TREE hierarchy, CHART datasets,

- Conversões de espaço de cor (HSV/HSL) — exigiriam aritmética não-trivial; se
  entrarem, têm de ser inteiras (zero-float).
