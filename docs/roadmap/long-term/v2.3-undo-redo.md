## Objetivo

Histórico determinístico de mutações por contexto. API genérica para qualquer widget. Coalescing automático para edições rápidas de texto.

## Escopo CapyUI

- Struct opaca:
  ```c
  struct capy_undo_stack;
  ```
- APIs:
  ```c
  int capy_undo_stack_init(struct capy_widget_context *ctx,
                           void *buf, uint32_t buf_size);
  int capy_undo_push(struct capy_widget_context *ctx,
                     const char *action_id,
                     const void *redo_data, uint32_t redo_len,
                     const void *undo_data, uint32_t undo_len);
  int capy_undo_undo(struct capy_widget_context *ctx);
  int capy_undo_redo(struct capy_widget_context *ctx);
  int capy_undo_can_undo(const struct capy_widget_context *ctx);
  int capy_undo_can_redo(const struct capy_widget_context *ctx);
  ```
- Coalescing: ações com mesmo `action_id` em janela de 500ms (configurável) são agrupadas.
- Storage limitado: buffer fornecido pelo caller; ao encher, descarta entries mais antigas (FIFO).
- Hook integrado: edições em TEXTBOX (v0.4) e RICH_TEXT (v2.1) registram automaticamente.

## Testes obrigatórios

1. **Push + undo + redo** retorna ao estado.
2. **Coalescing** agrupa 10 inserts de texto em 1 ação undo.
3. **Buffer cheio** descarta antigos sem corrupção.
4. **Undo em stack vazio** retorna `-1` sem crash.
5. **Determinismo:** mesma sequência → mesmo histórico.

## Notas

- Caller fornece undo_data/redo_data; CapyUI apenas armazena bytes.
- Aplicação real do undo é callback baseado em `action_id` (host re-aplica).
- Coalescing time threshold configurável via API.
