**Status:** planned (visão de longo prazo)
**ABI bump:** capy-ui-widget 2.x → **3.0 (quebra controlada)**
**Display-list schema:** **7 (atual) → 9**. As ops 3D opcionais (`CAPY_DL_LAYER_3D_PUSH`/`POP`) levam o schema a `9`; uma etapa 3.x intermediária pode introduzir o schema `8` antes. (Corrige uma versão anterior deste doc que dizia "8 → 9" — o schema corrente é `7`, ver `../contracts/abi-versions.md` e `src/widget/capy_display_list.h`.)
**Dependência:** **fase 2.x COMPLETA (2.0–2.13 entregues)** + 2.x em LTS ≥12m. (Anteriormente este doc citava "v2.5.0 entregue", de quando 2.5 era a fronteira.)
**Estimativa:** XL+ (múltiplas releases possíveis dentro do 3.0)
**Owner:** —

> **Pré-condição de deprecação (bloqueante para qualquer remoção em 3.0).**
> A `deprecation-policy.md` exige **≥2 minor releases** entre a marcação `CAPYUI_API_DEPRECATED` e a remoção num major. Hoje **nenhum símbolo 2.x está deprecado** (o único uso de `CAPYUI_API_DEPRECATED` no header é a definição da macro). Logo, a seção "Quebras em 3.0" abaixo **não pode** remover/renomear nada até que: (a) minors 2.x de deprecação (2.14+) marquem os símbolos-alvo por ≥2 releases, **ou** (b) um ADR de exceção seja registrado em `../tracking/DECISIONS.md`. Ver ADR-0006. Enquanto isso não acontecer, 3.0 só pode crescer aditivamente.

## Objetivo

Plataforma desktop multi-modal: voice input, AR/VR opcional, AI integration, acessibilidade WCAG 2.2 AA certificada, frame independence (1Hz–240Hz), crash-only design. **Mantém invariantes do núcleo (portátil, determinístico, sem deps externas).**

## Escopo CapyUI

### 1. Voice input contract

- Host fornece STT (speech-to-text); CapyUI roteia comando para foco atual.
- Eventos novos:
  - `CAPY_WIDGET_EVENT_VOICE_COMMAND` com texto reconhecido.
  - `CAPY_WIDGET_EVENT_VOICE_INTENT` com intent canônico (ex.: "submit", "cancel", "focus next").
- Widgets focáveis declaram intents aceitos via metadata aditiva.

### 2. AR/VR optional mode

- Display-list ganha ops 3D **opcionais**:
  - `CAPY_DL_LAYER_3D_PUSH`: vector translation 3D + rotation 3D.
  - `CAPY_DL_LAYER_3D_POP`.
- Compositor 2D **ignora** essas ops (degradação visual aceita).
- Compositor 3D (host com VR/AR) usa as ops para posicionar layers em espaço 3D.

### 3. AI integration hints

- Widgets podem expor intent:
  ```c
  void capy_widget_set_ai_intent(struct capy_widget *w,
                                 const char *intent_id,
                                 const char *context);
  ```
- Intents canônicos: `autocomplete`, `summarize`, `translate`, `explain`, `correct`.
- Host com LLM consome via callback; CapyUI não chama LLM diretamente.

### 4. Acessibilidade WCAG 2.2 AA certificada

- Auditoria formal de todas as primitivas.
- Contraste mínimo enforced em tokens.
- Foco visível em todos os widgets focáveis.
- Roles a11y cobrem 100% (já em v0.11).
- Relatório formal publicado em `docs/accessibility-audit-3.0.md`.

### 5. Multi-modal input com prioridade configurável

- Eventos pointer + key + touch + voice + gesture coexistem.
- API:
  ```c
  void capy_widget_context_set_input_priority(
      struct capy_widget_context *ctx,
      const enum capy_input_kind *priority_order,
      uint32_t count);
  ```
- Conflitos resolvidos em ordem; default: pointer > key > touch > voice > gesture.

### 6. Frame independence

- Modelo retido roda em qualquer refresh:
  - 1Hz (e-ink display).
  - 60Hz (desktop padrão).
  - 120Hz/240Hz (gaming displays).
- Animação adapta-se ao tick rate sem mudança de API.

### 7. Crash-only design

- Toda operação interrompida deixa estado consistente.
- Serialização v1.10 garante recuperação.
- Não há "graceful shutdown" — sempre se assume crash.

### Quebras em 3.0

- Remoções de APIs deprecadas em 2.x.
- Possíveis renomeações em headers.
- Guia de migração detalhado.

## Testes obrigatórios

1. **Voice command** roteado para widget focado.
2. **3D ops ignoradas** em compositor 2D sem crash.
3. **AI intent set** + callback recebe contexto correto.
4. **Auditoria WCAG** passa em fixtures.
5. **Multi-modal**: pointer + voice simultâneos respeitam prioridade.
6. **1Hz operation:** animação adapta-se sem perder transições.
7. **Crash recovery:** mata-processo no meio de mutação; deserialize recupera state.

## Notas

- Esta é a **visão final do roadmap atual**. Releases dentro de 3.x podem detalhar cada área.
- Invariantes inquebráveis preservados: zero deps, C11, determinístico.
- 2.x LTS continua por ≥12 meses pós-3.0.
- Ambicioso por design; cada subseção pode virar release dedicada (3.1 voice, 3.2 AR/VR, 3.3 AI, etc.) se necessário.
