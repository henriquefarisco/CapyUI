## Escopo entregue (widget core)

Plumbing primitives compartilhadas pelo file-manager app, save/open dialogs e qualquer shell que precise de taxonomia de toolbar ou truncation de breadcrumb. Widget core publica a math + as constantes; UX concreta vive em `src/apps/`.

### Macros (6)

- `CAPY_FILE_MGR_COMPACT_THRESHOLD_PX` (=600) — viewport < 600 px ativa modo compacto. Constante de referência; widget core não force.
- `CAPY_TOOLBAR_GROUP_NAV` (=0), `_VIEW` (=1), `_ACTION` (=2), `_LAYOUT` (=3), `_COUNT` (=4) — taxonomia estável dos 4 grupos da toolbar.

### Struct (1)

```c
struct capy_breadcrumb_segment {
  uint32_t id;
  uint16_t text_offset;  /* indexa text pool caller-owned */
  uint16_t text_len;
  uint16_t icon_image_id;
  uint8_t  is_clickable;
  uint8_t  reserved;
};
```

### 1 API determinística

```c
int capy_breadcrumb_truncate(
    const struct capy_breadcrumb_segment *in, uint32_t in_count,
    uint32_t available_width_px, uint32_t segment_avg_px,
    struct capy_breadcrumb_segment *out, uint32_t cap,
    uint32_t *out_count, uint8_t *out_dropdown);
```

Algoritmo:
1. `in_count == 0` → out_count=0, dropdown=0.
2. `in_count * avg <= available_width` → copy all, dropdown=0. (Fails se `cap < in_count`.)
3. Else: `max_inline = max(2, available_width / avg)`, clamp `min(cap, in_count)`. `out[0] = in[0]` (root), `out[1..N-1]` = tail de `in[]`. dropdown=1. Hosts renderizam o `…` entre `out[0]` e `out[1]`.

Determinístico, zero-float, sem alocação.

## Escopo desktop-session

Toolbar refatorada em 4 grupos via `CAPY_TOOLBAR_GROUP_*`, modo compacto < 600 px, breadcrumb dropdown usando `capy_breadcrumb_truncate`, status bar fixa. Vive em `src/apps/file_manager.c`.

## Testes entregues (8)

1. `breadcrumb_full_fit_no_dropdown` — 4×80=320 ≤ 800: todos inline.
2. `breadcrumb_truncated_root_plus_tail` — 8×100>300: root + 2 tail, dropdown=1.
3. `breadcrumb_min_two_with_narrow_width` — avg > available: floor em 2 segmentos.
4. `breadcrumb_empty_input` — in_count=0 → tudo zero.
5. `breadcrumb_single_segment` — 1 in → 1 out, dropdown=0.
6. `breadcrumb_null_guards` — todas as 6 combinações inválidas.
7. `breadcrumb_cap_too_small_for_full_fit` — cap=2 < in=4 com full-fit → -1.
8. `toolbar_group_taxonomy` — valores estáveis (NAV=0..COUNT=4) + threshold (=600).

## Decisões técnicas

- **Plumbing-only.** Widget core não dirige o file-manager; apenas publica math + taxonomy.
- **Determinístico.** Mesmo input → mesma saída byte-a-byte.
- **Floor em 2 segmentos.** Quando o budget é apertado, garantimos sempre root + current visíveis (UX coerente em telas estreitas).
- **Pre-flight cap check.** Em full-fit, `cap < in_count` retorna -1 antes de copiar (sem partial writes).
- **Reserved field em segment.** Forward-compat para flags futuras (icon-only mode, sticky-root, etc.).

## Deferido para minors futuros

- **Variable-width segment measurement.** Hoje 1 avg para a barra inteira; callback per-segment fica para futuro minor com text metrics.
- **Multi-line breadcrumb.** Política da desktop-session.
- **Locale-aware ellipsis.** RTL pode precisar de leading `…`; com v0.13.
- **Toolbar item descriptors.** Per-item icon/label/tooltip/shortcut ficam no desktop session.

## Objetivo

Melhorar a exibição dos botões da toolbar do gerenciador de arquivos: ícones consistentes, estados claros (hover/active/disabled), agrupamento lógico, atalhos visíveis e variantes compactas para janelas estreitas.

## Escopo desktop-session

- Toolbar do file manager (`src/apps/files.c` ou equivalente) refatorada em 4 grupos:
  1. **Navegação**: Back, Forward, Up, Reload.
  2. **Visualização**: Icons, List, Details, Tiles + dropdown "Sort by".
  3. **Ações**: New folder, Cut, Copy, Paste, Delete, Rename.
  4. **Layout**: Sidebar toggle, Preview pane toggle, Hidden files toggle.
- Cada botão: `image_id` (16×16 e 24×24 via DPI scope), label localizado, tooltip com keybind ("Ctrl+C — Copiar").
- Estados visuais via theme tokens (já existentes): `ACCENT_HOVER`, `ACCENT`, `DISABLED`, `FOCUS_RING`.
- Modo compacto: quando `bounds.w < 600`, grupos colapsam em overflow menu (`⋯`).
- Breadcrumb com dropdown clicável em cada segmento.
- Status bar fixa exibindo: contagem de items, soma de tamanhos da seleção, espaço livre do volume.

## Escopo CapyUI core

- Nenhum bump de ABI. Usa TABLE/TREE de 2.1 + theme tokens + DPI scope de 1.5.
