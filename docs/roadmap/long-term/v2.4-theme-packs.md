## Objetivo

Bundle `.captheme` com tokens, referências de fonts e ícones. Hot reload em dev. Marketplace-ready (host distribui assinados).

## Escopo CapyUI

- Formato `.captheme`:
  - Magic `"CTHM"`.
  - Versão (uint16).
  - Metadata: name, author, version, locale.
  - Tabela de tokens (`capy_token` enum → uint32).
  - Refs de fonts: nome canônico + variante (italic, bold).
  - Refs de ícones: id → bytes (PNG/SVG embarcado ou ref externa).
  - Assinatura opcional (Ed25519, processado pelo host).
- APIs:
  ```c
  int capy_theme_pack_load(const void *buf, uint32_t len,
                          struct capy_theme *out_theme,
                          struct capy_widget_context *ctx);
  int capy_theme_pack_validate(const void *buf, uint32_t len);
  ```
- Hot reload (dev mode): host CapyOS observa arquivo e re-aplica.

## Testes obrigatórios

1. **Pack válido carrega** todos os tokens.
2. **Pack corrompido rejeitado** com erro identificável.
3. **Versão futura** rejeitada limpo.
4. **Token desconhecido** ignorado controlamente.
5. **Round-trip:** apply → serialize → load → compare equivalent.

## Notas

- Assinatura opcional; CapyUI não valida (host CapyOS responsável).
- Fontes/ícones reais ficam no host; CapyUI armazena apenas refs (font_id, image_id).
- Sucessor evoluído de v0.14 theme serialization (que usa formato textual key=value).
