# v1.10.0 — Serialização do modelo retido

**Status:** delivered (CapyUI side, 2026-05-21) com escopo conservador: estrutura do tree + bounds + text + flags + tab_index são serializados; state mais profundo (text_edit, animações, theme tokens, locale, transform, DnD, IME, callbacks) **deferido** para fatia futura. Foundation suficiente para crash recovery e devtools snapshot.
**ABI bump:** capy-ui-widget 1.9 → 1.10 (aditivo)
**Display-list schema:** permanece **6** (serialização não toca o DL).
**Dependência:** v1.9.0 entregue
**Estimativa:** L (CapyUI ✅)
**Owner:** CapyUI

## Objetivo

Serializar e desserializar a árvore retida (estrutura + estado) em formato canônico. Permite restaurar formulários após crash, hot reload, snapshot para devtools.

## Escopo CapyUI

- Formato canônico (binário pequeno + textual debug):
  - Magic `"CUIS"` (CapyUI State).
  - Versão (uint16).
  - Árvore em pre-order: tipo, bounds, style, text, focus state, animations, text_edit state.
- APIs:
  ```c
  int capy_widget_serialize(const struct capy_widget *root,
                            void *out_buf, uint32_t cap,
                            uint32_t *out_len);

  int capy_widget_deserialize(const void *buf, uint32_t len,
                              struct capy_widget_context *ctx,
                              struct capy_widget **out_root);

  int capy_widget_serialize_text(const struct capy_widget *root,
                                 char *out, uint32_t cap);  /* debug */
  ```
- Serialização inclui:
  - Tipo e bounds.
  - Style (com tokens).
  - Text e text_edit state.
  - Focus state (focusable, tab_index).
  - Animation state (active animations).
  - Locale do contexto.
- **Não** inclui:
  - Ponteiros (`user_data`, callbacks) — devem ser re-conectados após deserialize.
  - Resources externos (font_id, image_id) — caller restaura.

## Testes que cobrem o contrato

- `test_state_serialize_header_and_magic` — magic `"CUIS"`, format version=1 LE, reserved=0, node_count=3.
- `test_state_round_trip` — serialize em ctx_a + deserialize em ctx_b produz árvore com mesmos type/bounds/text/focused/tab_index/child_count.
- `test_state_determinism` — mesma árvore serializada 2× produz bytes idênticos.
- `test_state_rejects_bad_magic_version_checksum` — corruption em magic, version, ou body byte cada um detectado como `-1`.
- `test_state_capacity_overflow_no_corruption` — buf < HEADER_SIZE: out_len=0, retorna -1; buf >= HEADER_SIZE mas < body: out_len >= 16, retorna -1; NULL guards.
- `test_state_text_dump` — produz formato `[type=N bounds=(x,y,w,h) flags=ve text="..." children=K]` indentado; capacity overflow retorna -1; NULL guards.
- `test_state_unknown_type_rejected` — type byte > `CAPY_WIDGET_TABS` (mesmo com checksum válido) rejeitado.

**Não testado em 1.10** (deferido para fatia futura):
- Versão antiga com upgrade automático — quando format_version > 1 existir.
- text_edit state, animações, theme tokens, locale, transform, DnD config, IME state.
- Callback pointers e user_data — documentados como caller-rewire.
- font_id, image_id e resources externos — documentados como caller-rebind.

## Notas

- Foundation para v2.5 (devtools time-travel debugging) e v2.3 (undo/redo).
- Formato binário explicitamente **little-endian** para portabilidade cross-host. Header de 16 bytes (magic + version + reserved + checksum + node_count). Per-node: type/flags/tab_index/bounds/text_len + text + child_count.
- Checksum FNV-1a 32-bit sobre o body (bytes [16, end)). Validado **antes** de qualquer allocação de widget em deserialize.
- Pre-order tree walk; child_count limitado a `CAPY_WIDGET_MAX_CHILDREN` (32); text_len limitado a `CAPY_WIDGET_MAX_TEXT - 1` (255). Limites enforced no deserialize.
- Deserialize allocates via `ctx->allocator`. Em falha mid-tree, todos os widgets criados são destruidos antes de retornar (no leak).
- O que **não** é preservado em 1.10: text_edit caret/seleção, animações ativas, theme tokens, locale, 2D transform, DnD config, IME state, layout cache, pool/slab state, callbacks (`on_click`/`on_change`/`on_submit`/`on_drop`), `user_data`, `font_id`, `image_id`. Hosts re-conectam após deserialize.
- Novo source file `src/widget/capy_widget_serialize.c` adicionado a `SRC_WIDGET` no Makefile.
