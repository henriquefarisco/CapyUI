## Objetivo

Serializar e desserializar a árvore retida (estrutura + estado) em formato canônico. Permite restaurar formulários após crash, hot reload, snapshot para devtools.

## Escopo CapyUI

- Formato canônico (binário pequeno + textual debug):
  - Magic `"CUIS"` (CapyUI State).
  - Versão (uint16).
  - Árvore em pre-order: tipo, bounds, style, text, focus state, animations, text_edit state.
- APIs:
  ```c
  int capy_widget_serialize(const struct capy_widget *root,
                            void *out_buf, uint32_t cap,
                            uint32_t *out_len);

  int capy_widget_deserialize(const void *buf, uint32_t len,
                              struct capy_widget_context *ctx,
                              struct capy_widget **out_root);

  int capy_widget_serialize_text(const struct capy_widget *root,
                                 char *out, uint32_t cap);  /* debug */
  ```
- Serialização inclui:
  - Tipo e bounds.
  - Style (com tokens).
  - Text e text_edit state.
  - Focus state (focusable, tab_index).
  - Animation state (active animations).
  - Locale do contexto.
- **Não** inclui:
  - Ponteiros (`user_data`, callbacks) — devem ser re-conectados após deserialize.
  - Resources externos (font_id, image_id) — caller restaura.

## Testes obrigatórios

1. **Round-trip:** serialize → deserialize produz árvore equivalente.
2. **Determinismo:** mesma árvore → mesmo blob byte-a-byte.
3. **Versão antiga (futuro)** deserializa com upgrade automático ou erro claro.
4. **Versão futura** rejeitada com erro identificável.
5. **Corruption (byte aleatório alterado)** detectada via checksum.
6. **Capacity overflow** retorna -1 sem corromper.
7. **Text mode** produz dump legível para debugging.

## Notas

- Foundation para v2.5 (devtools time-travel debugging).
- Foundation para v2.3 (undo/redo).
- Formato binário, com versão e magic; deserialize valida ambos antes de tocar dados.
