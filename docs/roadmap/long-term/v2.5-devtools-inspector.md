## Escopo entregue

### Tree flatten

API determinística que faz depth-first pre-order do widget tree e popula um array flat de nodes + um text pool, ambos caller-owned. Ordem espelha `capy_widget_emit_recursive` (estável cross-run, alinhada com a emissão do DL).

```c
struct capy_inspector_node {
  uint32_t id;
  uint16_t type;          /* uint16 do enum capy_widget_type */
  uint16_t depth;
  uint32_t parent_index;  /* CAPY_INSPECTOR_NO_PARENT (=0xFFFFFFFFu) na root */
  uint32_t child_count;
  int32_t x, y, w, h;
  uint32_t flags;         /* CAPY_INSPECTOR_FLAG_* bitmask */
  int16_t  tab_index;
  uint16_t font_id;
  uint16_t text_offset;   /* posição no text_pool */
  uint16_t text_len;
  uint32_t layout_version;
  uint32_t dirty_version;
};

int capy_widget_inspect(const struct capy_widget *root,
                        struct capy_inspector_node *out_nodes,
                        uint32_t cap, char *out_text_pool,
                        uint32_t text_cap, uint32_t *out_node_count,
                        uint32_t *out_text_used);
```

Flags publicadas: `VISIBLE`, `ENABLED`, `FOCUSED`, `FOCUSABLE`, `CHECKED`, `HOVERED`, `LAYOUT_DIRTY`, `HAS_TRANSFORM`, `HAS_VIRTUAL_SRC`, `HAS_DRAG_PAYLOAD`, `HAS_DROP_TARGET`.

### Perf counters snapshot

```c
struct capy_perf_counters {
  uint32_t widget_count;
  uint32_t plugin_count;
  uint32_t undo_count;
  uint32_t undo_redo_count;
  uint32_t undo_capacity;
  uint16_t dpi_scale_x256;
  uint16_t reserved_dpi;
  uint32_t frame_budget_microseconds;
  uint16_t theme_variant;
  uint8_t  theme_high_contrast;
  uint8_t  reserved[1];
};

int capy_perf_counters_snapshot(const struct capy_widget_context *ctx,
                                const struct capy_widget *root,
                                struct capy_perf_counters *out);
```

`root == NULL` é tolerado (widget_count = 0). Undo fields ficam zero quando o stack não está attach.

### Fail-closed

- NULL `root`/`out_nodes`/`out_node_count`/`out_text_used` → -1.
- `text_cap > 0` com `out_text_pool == NULL` → -1.
- Capacity overflow ou text pool overflow → -1, rollback de `*out_node_count` e `*out_text_used` para 0.
- Mutações zero: as duas APIs são read-only sobre o tree e ctx.

## Testes entregues (7)

1. `inspector_root_only_returns_one_node` — single node, depth/parent/flags corretos.
2. `inspector_depth_first_preorder` — tree (A(B(C),D)) → ordem [A,B,C,D] com depths [0,1,2,1] e parent_index corretos.
3. `inspector_text_pool_copies_label_text` — bytes copiados sem NUL, `text_offset`/`text_len` corretos.
4. `inspector_capacity_overflow_fails_closed` — cap=2, 3 nodes → -1 + rollback.
5. `inspector_text_pool_overflow_fails_closed` — text_cap=2 com label "long-text" → -1 + rollback.
6. `inspector_null_guards` — 5 NULL-arg patterns rejeitados.
7. `perf_counters_snapshot_populates_fields` — widget_count, plugin/undo/dpi/frame_budget/theme; NULL ctx/out rejeitado; NULL root tolerado.

## Deferido para minors futuros

- **`capy_devtools_dump_display_list`** — emit textual canônico por op (útil para snapshot tests do DL, diff em git).
- **`capy_devtools_dump_tree`** — versão textual do flatten (formato custom; sem dep de JSON).
- **Event recording / replay** — ring buffer caller-owned de `(timestamp_ms, capy_widget_event)` com `_replay` que re-dispatcha o stream. Constrói sobre undo stack 2.3 + state serialization 1.10.
- **Headless harness CLI** — `capy-headless` tool (fora do core portável) que carrega state, alimenta event stream e captura DL para comparar com golden.
- **IDE protocol** — wire format textual para integração com Windsurf / VSCode.

## Notas

- **Zero allocation.** Ambas as APIs escrevem em buffers caller-owned.
- **Read-only.** Nenhuma mutação no tree, ctx, ou theme.
- **Determinístico.** Mesmo `(root, ctx)` → mesmo flatten + mesmo snapshot.
- **DL schema permanece 7.**
- Compõe com v1.10 serialize (snapshot offline) + v2.3 undo (history) + v2.4 theme packs (visual replay) para futura time-travel UI.
