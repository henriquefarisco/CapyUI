## Objetivo

Ferramentas de inspeção do modelo retido e display-list. Time-travel debugging. Headless harness para CI de apps.

## Escopo CapyUI

- Inspector API:
  ```c
  int capy_devtools_dump_tree(const struct capy_widget *root,
                              char *out, uint32_t cap);   /* JSON-like */
  int capy_devtools_dump_display_list(const struct capy_display_list *dl,
                                      char *out, uint32_t cap);
  int capy_devtools_record_session(struct capy_widget_context *ctx,
                                   void *buf, uint32_t cap);
  int capy_devtools_replay_session(const void *buf, uint32_t len,
                                   struct capy_widget_context *ctx);
  ```
- Time-travel debugging:
  - Recording: cada `dispatch_*` registra timestamp + payload + state diff.
  - Replay: re-aplica eventos em ordem determinística.
- Headless harness:
  - CLI `capy-headless` (futuro tool) que carrega state, aplica events, captura outputs.
  - Útil para integration tests de apps sem GUI.

## Testes obrigatórios

1. **Dump JSON-like** parseable por consumidores externos.
2. **Record + replay** produz mesmo estado final.
3. **Replay de session de versão antiga** funciona (versionado).
4. **Headless harness** executa fixture de teste sem display real.

## Notas

- Dump é texto canônico (não JSON real para evitar dep; formato custom documentado).
- Time-travel usa v1.10 serialization como base.
- Sucessor pode integrar a IDEs (Windsurf, VSCode) via protocolo simples.
