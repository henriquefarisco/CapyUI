## Objetivo

Propagação de dirty para subárvore + diff incremental do display-list + frame budget para o host.

## Escopo CapyUI

- API:
  ```c
  void capy_widget_invalidate(struct capy_widget *w);
  void capy_widget_invalidate_subtree(struct capy_widget *w);
  uint32_t capy_widget_dirty_version(const struct capy_widget *w);
  int capy_display_list_diff_incremental(
      const struct capy_display_list *prev,
      const struct capy_display_list *next,
      struct capy_ui_rect *out_dirty,
      uint32_t out_cap);
  void capy_widget_frame_budget(struct capy_widget_context *ctx,
                                uint32_t max_microseconds);
  ```
- Cada widget ganha `uint32_t dirty_version` monotônico.
- `emit` consulta `dirty_version` e pode pular subárvores não-modificadas (com cache).
- Frame budget: host avisa CapyUI sobre tempo disponível; CapyUI pode degradar (skip non-essential ops).

## Testes obrigatórios

1. **Invalidate em folha propaga para root** corretamente.
2. **Diff incremental** é O(visíveis modificados), não O(árvore inteira).
3. **Frame budget excedido** documentado em log/callback.
4. **Cache de emit** produz mesma display-list após invalidate seletivo.

## Contratos atualizados

- `contracts/display-list.md` (diff incremental documentado).
- `contracts/widget-model.md` (dirty_version aditivo).

## Notas

- Aditivo. Programs sem `invalidate` continuam funcionando (tudo "dirty" implicitamente).
- Frame budget é hint, não enforcement; CapyUI pode ignorar se necessário.
