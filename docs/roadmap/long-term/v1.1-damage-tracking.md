# v1.1.0 — Damage tracking otimizado

**Status:** delivered (CapyUI side, 2026-05-21). Benchmark suite / frame-budget enforcement seguem como opt-in para CI externo.
**ABI bump:** capy-ui-widget 1.0 → 1.1 (aditivo — primeiro minor pós-freeze)
**Display-list schema:** 4 (sem mudança)
**Dependência:** v1.0.0 entregue. Não bloqueada.
**Estimativa:** M (CapyUI ✅) + benchmark CI (futuro)
**Owner:** CapyUI

## Objetivo

Propagação de dirty para subárvore + diff incremental do display-list + frame budget para o host.

## Escopo CapyUI (entregue)

- API pública (em `src/widget/capy_widget.h`):
  ```c
  void capy_widget_invalidate(struct capy_widget *w);              /* since 0.15, bumps dirty_version since 1.1 */
  void capy_widget_invalidate_subtree(struct capy_widget *w);      /* since 1.1 */
  uint32_t capy_widget_dirty_version(const struct capy_widget *w); /* since 1.1 */
  int capy_display_list_diff_incremental(                          /* since 1.1 */
      const struct capy_display_list *prev,
      const struct capy_display_list *next,
      struct capy_ui_rect *out_dirty,
      uint32_t out_cap);
  void capy_widget_frame_budget(struct capy_widget_context *ctx,   /* since 1.1 */
                                uint32_t microseconds);
  ```
- Campo aditivo `uint32_t dirty_version` no tail de `capy_widget` (since 1.1) — monotônico, bumpado por `capy_widget_invalidate` e `capy_widget_invalidate_subtree`. Overflow wraps silently; hosts amostram delta entre snapshots, não valor absoluto.
- Campo aditivo `uint32_t frame_budget_microseconds` no tail de `capy_widget_context` (since 1.1). Seeded em 0 (sem hint). `capy_widget_frame_budget(ctx, n)` armazena `n`; CapyUI **não** enforce — é hint cooperação.
- `capy_display_list_diff_incremental`: skip de prefixo comum determinístico antes de rodar o diff posicional. **Byte-identical** ao `capy_widget_diff` (since 0.8) na saída; apenas mais rápido quando o prefixo é longo. Suffix skip seria semanticamente diferente (re-aligna prev/next e descarta trailing rects que `capy_widget_diff` emite por design) — deliberadamente não aplicado para preservar o contrato.
- `capy_widget_emit` não consulta `dirty_version` ainda — a infraestrutura está disponível para uma fatia futura (cache de emit) sem nova quebra de ABI.
- Frame budget enforcement (callback, log, throttling) fica para slice futura quando integrar com input plumbing e animation.

## Testes que cobrem o contrato

- `test_invalidate_bumps_dirty_version` — `capy_widget_invalidate` bumpa `dirty_version` em todos os ancestrais visitados (além de preservar o `layout_dirty=1` propagado desde 0.15).
- `test_invalidate_subtree_walks_down` — `capy_widget_invalidate_subtree` bumpa `dirty_version` em `widget` e em todos os descendentes, **sem** tocar os ancestrais (diferente de `capy_widget_invalidate`).
- `test_dirty_version_accessor` — leitor retorna o valor atual; bumpa em cada `invalidate`; NULL → 0.
- `test_frame_budget_round_trip` — setter armazena no campo do contexto; NULL é no-op.
- `test_diff_incremental_matches_diff_identical` — prev=next → 0 rects em ambos.
- `test_diff_incremental_matches_diff_trailing_change` — sequência byte-idêntica ao `capy_widget_diff` quando só um cmd no meio difere.
- `test_diff_incremental_overflow_returns_minus_one` — overflow de cap retorna `-1`; NULL out com cap>0 → `-1`; ambos NULL e cap>0 → 0 (sem-op deterministicamente).
