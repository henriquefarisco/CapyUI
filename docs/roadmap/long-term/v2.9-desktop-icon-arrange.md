**Status:** delivered (2026-05-21)
**ABI bump:** capy-ui-widget 2.8 → 2.9 (aditivo: blob format) + capy-ui-desktop-session 1 (aditivo, sem bump)
**Display-list schema:** 7 (sem mudança)

**Estimativa:** M

## Escopo entregue (widget core)

Plumbing primitives + serializer/parser para o blob `CDLA` que o desktop session persiste em `/var/state/capyui/desktop-layout.bin`. Widget core não dirige o drag, não toca disco — apenas detém as struct shapes e o round-trip canônico.

### Macros (7+5+2)

- `CAPY_DESKTOP_LAYOUT_MAGIC0..3` = `'C','D','L','A'`.
- `CAPY_DESKTOP_LAYOUT_FORMAT_VERSION` (=1).
- `CAPY_DESKTOP_LAYOUT_HEADER_SIZE` (=24).
- `CAPY_DESKTOP_LAYOUT_ENTRY_SIZE` (=16).
- `CAPY_DESKTOP_SORT_*`: MANUAL/NAME/TYPE/MTIME/SIZE + COUNT (=5).
- `CAPY_DESKTOP_LAYOUT_FLAG_ALIGN_RIGHT` (=0x01), `_VERTICAL_FIRST` (=0x02).

### Structs

`capy_desktop_layout { cell_w, cell_h, snap, auto_arrange, sort_by, flags }`. `capy_desktop_icon_position { icon_id, x, y, grid_x, grid_y, pinned, reserved[3] }`.

### Blob canonical LE (header 24B + entries 16B)

```
0   4  magic = "CDLA"
4   2  format_version (=1)
6   1  snap
7   1  auto_arrange
8   4  FNV-1a checksum sobre body
12  2  cell_w        (> 0)
14  2  cell_h        (> 0)
16  1  sort_by       (< SORT_COUNT)
17  1  flags
18  2  entry_count
20  4  reserved (=0)
24+ N×16B  per-entry: uint32 icon_id; int16 x, y, grid_x, grid_y; uint8 pinned; uint8 reserved[3]
```

### 3 APIs

- `capy_desktop_layout_validate(buf, len)` — magic + version + reserved + length + checksum + cell>0 + snap/auto≤1 + sort<COUNT.
- `capy_desktop_layout_load(buf, len, *out_layout, *out_positions, cap, *out_count)` — valida + escreve config + copia entries até `cap`. `*out_count` = declared count (short-buffer detection via `*out_count > cap`).
- `capy_desktop_layout_serialize(*layout, *positions, count, *out, cap)` — escreve blob canônico byte-determinístico; retorna bytes escritos ou -1.

## Decisões técnicas

- **Plumbing-only.** Widget core nunca abre arquivo, nunca dirige drag, nunca decide posição.
- **FNV-1a sobre body.** Mesma função que `CUIS`/`CTHM` — audit comum.
- **Reserved fail-closed.** Header + per-entry reserved devem ser zero.

## Deferido para minors futuros

- **Multi-monitor envelope.** Hoje 1 blob = 1 monitor.
- **Group folders.** v2.11 icon system.
- **Locale-aware sorting.** Surface v0.13 locale.
- **Drag hit-test no core.** Vive no desktop-session.

## Objetivo

Permitir que o usuário mova ícones do desktop por drag, com snap-to-grid configurável, posições persistidas entre sessões e auto-arrange opcional.

## Escopo desktop-session

- Cada `desktop_icon` ganha campos `grid_x`/`grid_y` (uint16) + `pinned` (uint8).
- `struct capy_desktop_layout { uint16_t cell_w, cell_h; uint8_t snap, auto_arrange, sort_by; uint8_t flags; }`:
  - `sort_by`: 0=manual, 1=name, 2=type, 3=mtime, 4=size.
  - `flags`: bit0 = align grid à esquerda, bit1 = à direita, bit2 = vertical primeiro.
- Roteamento de DRAG_BEGIN/MOVE/END nos ícones do desktop (handler em `src/desktop/`):
  - DRAG_MOVE: emite `CAPY_DL_DRAG_PREVIEW` (quando 1.6 follow-up entregar) ou render fantasma via overlay layer.
  - DRAG_END: se `snap=1`, encaixa em `(round(x/cell_w), round(y/cell_h))`; senão posição exata.
- Persistência: `capy_widget_serialize` (1.10) escreve um blob `/var/state/capyui/desktop-layout.bin` com magic `"CDLA"` versionado.
- Context menu do desktop ganha submenu "Arrange icons by..." (name/type/mtime/size) e toggle "Auto arrange" + "Align to grid".

## Escopo CapyUI core

- Nenhum bump da ABI widget. Apenas o desktop-session usa os campos `grid_x/grid_y` e dispara DRAG_END.

## Testes obrigatórios (host-side, em `tests/test_desktop_layout.c`)

1. **Drag move single icon** persiste nova posição.
2. **Snap-to-grid** arredonda para cell mais próxima.
3. **Auto arrange** por nome ordena alfabeticamente A→Z.
4. **Auto arrange** por mtime descendente.
5. **Pinned icons** ignoram auto-arrange.
6. **Persist round-trip** após reboot simulado restaura layout exato.
7. **Collision**: dois ícones na mesma cell → segundo desloca para próxima livre.

## Notas

- Coordenadas em pixels; `grid_x/grid_y` são derivados.
- "Show desktop" / esconder/mostrar não rearranja posições.
- Migração: layout legado sem `CDLA` carrega como `sort_by=name`, auto-arrange off.
