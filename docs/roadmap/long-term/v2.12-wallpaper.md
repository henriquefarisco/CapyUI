**Status:** delivered (2026-05-21)

**Display-list schema:** 7 (sem mudança)

## Escopo entregue (widget core)

Plumbing primitives + serializer/parser para o blob `CWLP` que o desktop session persiste em `/var/state/capyui/wallpaper.bin` (global ou per-user). Widget core não compõe wallpaper, não agenda timer de slideshow, não toca disco — apenas detém as struct shapes e o round-trip canônico.

### Macros (13)

- 4 magic bytes: `CAPY_WALLPAPER_MAGIC0..3` = `'C','W','L','P'`.
- `CAPY_WALLPAPER_FORMAT_VERSION` (=1), `_HEADER_SIZE` (=24), `_ENTRY_SIZE` (=8).
- 6 fit modes: `CAPY_WALLPAPER_FIT_STRETCH/FILL/FIT/CENTER/TILE` + `_COUNT` (=5).
- 2 flags: `CAPY_WALLPAPER_FLAG_RANDOM` (=0x01), `_PER_MONITOR` (=0x02).

### Structs

```c
struct capy_wallpaper_config {
  uint32_t default_image_id;
  uint16_t slideshow_interval_sec;
  uint16_t monitor_index;     /* 0 = global, >= 1 per-monitor */
  uint8_t  fit_mode;
  uint8_t  flags;
  uint16_t reserved;
};

struct capy_wallpaper_slide {
  uint32_t image_id;          /* >= 1 */
  uint16_t duration_sec;      /* >= 1 */
  uint16_t reserved;
};
```

### Blob canonical LE (header 24B + slides 8B)

```
0   4  magic = "CWLP"
4   2  format_version (=1)
6   1  fit_mode        (< FIT_COUNT)
7   1  flags
8   4  FNV-1a checksum sobre body
12  4  default_image_id
16  2  slideshow_interval_sec
18  2  slide_count
20  2  monitor_index
22  2  reserved (=0)
24+ N×8B  per-slide: uint32 image_id (>0); uint16 duration_sec (>0); uint16 reserved (=0)
```

### 3 APIs

- `capy_wallpaper_validate(buf, len)` — magic + version + reserved (header + per-entry) + fit_mode + length + checksum + per-slide `image_id > 0` + `duration_sec > 0`.
- `capy_wallpaper_load(buf, len, *out_config, *out_slides, cap, *out_count)` — valida + escreve config + copia slides até `cap`. Short-buffer detection via `*out_count > cap`.
- `capy_wallpaper_serialize(*config, *slides, count, *out, cap)` — pre-flight valida fit_mode + per-slide ids/durations + cap; escreve blob canônico byte-determinístico.

## Escopo desktop-session

Settings → Personalization app + slideshow timer + wallpaper composite ficam em `src/apps/settings_wallpaper.c` e no compositor. Per-user persistence via envelope ao redor do blob (host).

## Testes entregues (7)

1. `wallpaper_roundtrip` — config com 3 slides; todos campos preservados.
2. `wallpaper_no_slides_roundtrip` — count=0; 24-byte header só.
3. `wallpaper_invalid_fit_rejected` — `fit_mode >= COUNT` → -1 pre-flight.
4. `wallpaper_slide_zero_id_rejected` — slide com `image_id=0` ou `duration_sec=0` → -1.
5. `wallpaper_bad_magic_and_checksum_rejected` — magic alterada + flip byte sem re-checksum → -1.
6. `wallpaper_short_buffer_reports_count` — cap=2 < 6 declared; out_count=6.
7. `wallpaper_null_guards` — combinações NULL.

## Decisões técnicas

- **Plumbing-only.** Widget core nunca toca pixels, nunca agenda timer, nunca lê arquivo.
- **FNV-1a + reused helpers.** Reusa byte-helpers e FNV-1a do `CDLA` (v2.9). Format consistency com `CUIS`/`CTHM`/`CDLA`.
- **Per-slide guard.** `image_id=0` rejeitado (reservado v2.11 silhouette). `duration_sec=0` rejeitado (sem slide instantâneo).
- **monitor_index** uint16 — `0 = global`, `≥ 1 = per-monitor`. Compõe com v2.6 display mode (monitor index do controller).
- **Reserved fail-closed** em ambos header (offset 22) e per-entry (offset 6).

## Deferido para minors futuros

- **Live wallpaper / video.** `FLAG_LIVE_HANDLE` + callback no desktop session.
- **Per-locale defaults.** Surface v0.13 locale.
- **Transition effects.** Cross-fade / slide / dissolve — política desktop-session animation.
- **Image format negotiation.** Decode/scaling via v2.11 icon provider; widget core só carrega `image_id` opaco.

## Objetivo

Trocar wallpaper via UI, com modos de fit (stretch/fill/fit/center/tile), slideshow opcional, wallpaper por usuário e por monitor.

## Escopo desktop-session

- Struct `capy_wallpaper_config { uint32_t image_id; uint8_t fit_mode; uint8_t per_monitor; uint16_t monitor_index; uint32_t slideshow_interval_s; uint8_t shuffle; }`.
- `fit_mode`: 0=stretch, 1=fill (crop), 2=fit (letterbox), 3=center, 4=tile.
- Per-user: arquivo `/var/state/capyui/wallpaper-<uid>.bin` (magic `"CWLP"`).
- Per-monitor: array de configs indexado por `monitor_index` do v1.5.
- Slideshow: timer no dispatcher; após `slideshow_interval_s`, troca para próxima image_id de uma playlist.
- App **Settings → Appearance → Wallpaper**:
  - Grid de thumbnails (usa v2.11 thumbnail system).
  - Dropdown fit mode.
  - Toggle "Different wallpaper per monitor".
  - Slideshow controls (folder picker, interval, shuffle).
- Right-click no desktop ganha "Set as wallpaper" para imagens.

## Escopo CapyUI core

- Nenhum bump. Tudo no desktop-session usando v2.11 + v1.10 serialize.

## Testes obrigatórios (host-side)

1. **Set wallpaper** persiste por usuário.
2. **Fit modes** todos os 5 calculam destino correto vs source.
3. **Per-monitor** independent configs por `monitor_index`.
4. **Slideshow** avança após `interval_s` simulado.
5. **Shuffle** não repete enquanto playlist tem >1 item.
6. **Migration** wallpaper legado (image_id direto sem CWLP) carrega como `fit=fill, per_monitor=0`.
7. **Multi-user**: troca de usuário no login restaura wallpaper específico.
