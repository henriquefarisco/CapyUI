# v1.8.0 — IME rico

**Status:** planned
**ABI bump:** capy-ui-widget 1.7 → 1.8 (aditivo)
**Display-list schema:** 6 (sem mudança)
**Dependência:** v1.7.0 entregue + v0.4 (text edit)
**Estimativa:** L
**Owner:** —

## Objetivo

Modelo IME completo: candidate window, composition state expandido, contratos para CJK/Hangul/Indic. Host fornece engine real.

## Escopo CapyUI

- Struct:
  ```c
  struct capy_ime_state {
    const char *preedit;       /* texto em composição */
    uint16_t preedit_len;
    uint16_t cursor_pos;       /* posição no preedit */
    const char **candidates;   /* lista de candidatos */
    uint16_t candidate_count;
    uint16_t selected_index;
    uint8_t composition_phase; /* 0=idle, 1=composing, 2=selecting */
    uint8_t reserved[3];
  };
  ```
- APIs:
  - `void capy_ime_set_preedit(struct capy_widget *w, const char *text, uint16_t len, uint16_t cursor);`
  - `void capy_ime_set_candidates(struct capy_widget *w, const char **list, uint16_t count, uint16_t selected);`
  - `void capy_ime_commit(struct capy_widget *w, const char *final_text, uint16_t len);`
  - `void capy_ime_cancel(struct capy_widget *w);`
- Eventos abstratos:
  - `CAPY_WIDGET_EVENT_IME_COMPOSE_START`
  - `CAPY_WIDGET_EVENT_IME_PREEDIT_UPDATE`
  - `CAPY_WIDGET_EVENT_IME_COMMIT`
  - `CAPY_WIDGET_EVENT_IME_CANCEL`
- Candidate window renderizada via display-list (pequeno popup com lista de candidatos).

## Escopo CapyOS

- Engine IME real (mozc, anthy, Hangul-engine, ou subset próprio).
- Bridge eventos do IME → APIs CapyUI acima.

## Testes obrigatórios

1. **CJK composition:** preedit aparece, candidate window mostra opções, commit insere texto final.
2. **Hangul jamo combination:** ㄱ + ㅏ + ㅇ → 강 commitado.
3. **Cancel** preserva texto pré-composição.
4. **Candidate selection** via setas atualiza `selected_index`.
5. **Multi-candidate** rendering correto no display-list.
6. **Determinismo do modelo** (engine real é separado).

## Notas

- Modelo IME no core; engine real no host.
- v0.4 IME stub continua funcionando para casos sem composição rica.
- Memória de candidates: caller fornece lista; CapyUI só armazena ponteiros.
