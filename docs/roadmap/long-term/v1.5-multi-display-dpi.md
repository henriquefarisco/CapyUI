# v1.5.0 — Multi-display e DPI scaling

**Status:** planned
**ABI bump:** capy-ui-widget 1.4 → 1.5 (aditivo)
**Display-list schema:** 5 → 6 (`CAPY_DL_DPI_SCOPE`)
**Dependência:** v1.4.0 entregue
**Estimativa:** L
**Owner:** —

## Objetivo

Suporte a múltiplos displays com DPI diferentes (1x, 1.5x, 2x, 3x). Layout fluido; snap em grid de pixel inteiro.

## Escopo CapyUI

- API:
  ```c
  void capy_widget_set_dpi_scale(struct capy_widget_context *ctx,
                                 uint16_t scale_x256);
  ```
  - `scale_x256 = 256` → 1.0× (default).
  - `scale_x256 = 384` → 1.5×.
  - `scale_x256 = 512` → 2.0×.
- Layout multiplica `padding`, `gap`, `min_w/h`, `max_w/h` por `scale_x256 / 256` (em fixed-point).
- Snap em pixel inteiro após escala (evita sub-pixels).
- Op aditiva `CAPY_DL_DPI_SCOPE` no display-list (schema → 6):
  - `rect`: região afetada.
  - `font_size`: ajustado para o DPI.
  - Compositor honra ou ignora conforme suporte.
- API:
  ```c
  uint16_t capy_widget_get_dpi_scale(const struct capy_widget_context *ctx);
  ```

## Testes obrigatórios

1. **Scale 256 = comportamento atual** (compatibilidade).
2. **Scale 512** dobra dimensões de padding/gap/bounds.
3. **Snap em pixel inteiro:** `width = 100 * 1.5 = 150` (exato), não `149.99`.
4. **DPI por widget tree** (multi-display): cada `capy_widget_context` independente.
5. **Display-list reflete scale** via `DPI_SCOPE` em hosts capazes.

## Notas

- Aditivo. Hosts sem multi-DPI usam scale 256.
- Compositor CapyOS lê `DPI_SCOPE` para escolher rasterizer adequado.
- Fontes (v0.9 metrics hook) recebem DPI atualizado.
