# v2.8.0 — Contrast & accessibility theme refinement

**Status:** delivered (2026-05-21)
**ABI bump:** capy-ui-widget 2.7 → 2.8 (aditivo)
**Display-list schema:** 7 (sem mudança)
**Dependência:** v2.4 theme packs
**Estimativa:** S

## Escopo entregue

- 2 macros de threshold publicadas: `CAPY_CONTRAST_AA_X1000` (=4500), `CAPY_CONTRAST_AAA_X1000` (=7000). Variant macro `CAPY_THEME_VARIANT_DARK_HIGH_CONTRAST` (=3).
- `struct capy_contrast_finding { uint8_t fg_token; uint8_t bg_token; uint16_t ratio_x1000; uint8_t passes_aa; uint8_t passes_aaa; uint8_t reserved[2]; }`.
- 3 APIs:
  - `uint32_t capy_theme_contrast_ratio_x1000(uint32_t fg, uint32_t bg)` — WCAG-2.1-style ratio × 1000, clamp `[1000, 21000]`. Alpha ignorado.
  - `int capy_theme_audit_wcag(theme, out, cap)` — caminha os pares canônicos (11 pares: FG_PRIMARY/BG_BASE, FG_PRIMARY/BG_RAISED, FG_MUTED/BG_BASE, FG_INVERSE/ACCENT, BORDER_FOCUS/BG_BASE, FOCUS_RING/BG_BASE, DANGER/BG_BASE, WARNING/BG_BASE, SUCCESS/BG_BASE, INFO/BG_BASE, DISABLED/BG_BASE) e escreve um `capy_contrast_finding` por par. `cap == 0` é dry-run (retorna count total).
  - `struct capy_theme capy_theme_default_dark_high_contrast(void)` — 4ª factory built-in, todas as pairs ≥ 7:1 (AAA).
- Cálculo zero-float: linearização sRGB → Q16.16 via `lin = c * c * 65536 / 65025` (γ≈2.0 approximation), weighted luminance `L = (2126*R + 7152*G + 722*B) / 10000`. Ratio `(L_max + 3277)*1000 / (L_min + 3277)` (offset 0.05 ≈ 3277).

## Decisões técnicas

- **Zero-float.** Cálculo inteiro 64-bit. Determinístico cross-platform.
- **γ≈2.0 approximation** em vez da fórmula WCAG-2.1 exata (`((c+0.055)/1.055)^2.4`). Diferenças bounded e ambas concordam nos AA/AAA pass/fail dos pares canônicos. Documentado no header como compromisso de simplicidade.
- **Audit pairs** estáticos no source (não data-driven) — auditável + lineage-trackable.
- **`uint16_t ratio_x1000`** no finding cabe em 21000 < 65535.
- **Dry-run `cap=0`** permite hosts sizing o buffer com 1 chamada.

## Testes entregues (8)

1. `contrast_white_on_black_is_21000` — ratio máximo + simétrico.
2. `contrast_same_color_is_1000` — ratio mínimo (1.0).
3. `contrast_alpha_ignored` — high alpha vs low alpha = mesmo RGB → mesmo ratio.
4. `contrast_ratio_in_bounds` — mid-grey on black ∈ (1000, 21000).
5. `audit_dry_run_returns_total` — `cap=0` retorna pair count.
6. `audit_high_contrast_all_pass_aa` — high_contrast theme: todos pairs ≥ 4500.
7. `audit_dark_high_contrast_all_pass_aaa` — novo theme: todos pairs ≥ 7000 + variant byte correto.
8. `audit_null_and_capacity_overflow` — NULL theme/out + cap < total.

## Deferido para minors futuros

- **WCAG 3 APCA.** Perceptual contrast (Lc score) — novo set de APIs + tabelas.
- **Large-text AA threshold (3.0).** Por-par size metadata.
- **Per-locale palette overrides.** Locale-aware theming.
- **Auto-fix suggestions.** Audit findings hoje só reportam — não sugerem ajustes.

## Objetivo

Refinar a paleta dos 3 temas built-in (`default_light`, `default_dark`, `high_contrast`) para garantir WCAG 2.1 AA (4.5:1 em texto normal, 3:1 em texto grande) em todos os pares foreground/background usados pelos widgets. Adicionar API de auditoria de contraste runtime.

## Escopo

- Reavaliação de cada par token f/g em `src/widget/capy_theme.c`:
  - `BG_BASE/FG_PRIMARY`, `BG_RAISED/FG_PRIMARY`, `BG_SUNKEN/FG_MUTED`, `ACCENT/FG_INVERSE`, `BORDER_FOCUS/BG_BASE`, etc.
  - Ajustes pontuais nos valores hex para fechar gaps onde a razão atual está entre 3:1 e 4.5:1.
- Novo tema built-in `default_dark_high_contrast` (4º factory) — variant escuro com paridade 7:1.
- API nova:
  ```c
  uint32_t capy_theme_contrast_ratio_x1000(uint32_t fg, uint32_t bg);
  int capy_theme_audit_wcag(const struct capy_theme *t,
                            struct capy_contrast_finding *out, uint32_t cap);
  ```
  `capy_contrast_ratio_x1000` retorna a razão WCAG multiplicada por 1000 (Q0; inteiro).
  `audit_wcag` percorre os pares conhecidos e popula findings em buffer caller-owned.
- Struct `capy_contrast_finding { uint8_t fg_token, bg_token; uint16_t ratio_x1000; uint8_t passes_aa, passes_aaa; }`.

## Testes obrigatórios

1. **Built-in themes**: cada par f/g declarado passa AA (ratio ≥ 4500).
2. **`high_contrast`** passa AAA (ratio ≥ 7000) em todos os pares.
3. **`contrast_ratio_x1000(#FFFFFF, #000000) == 21000`**.
4. **`contrast_ratio_x1000(x, x) == 1000`** (mesma cor).
5. **Audit** popula findings determinísticos no buffer.
6. **Audit capacity 0** retorna total sem escrever.
7. **Determinismo cross-platform** do cálculo (relative luminance Q24.8).

## Notas

- Cálculo segue WCAG 2.1: `L = 0.2126*R + 0.7152*G + 0.0722*B`, channels linearizados via tabela 256-entry (zero float).
- Tokens cuja razão piora pós-ajuste **não** são quebra de ABI (valores hex são internos); semver patch elegível, mas escolhemos minor para sinalizar mudança visual.
- App **Settings → Appearance** ganha botão "Run contrast audit" que chama a API e exibe findings num TABLE.
